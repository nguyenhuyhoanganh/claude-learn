# Bài 4: AWS Free Tier — đăng ký, root user, IAM user, MFA bảo mật

AWS là cloud provider lớn nhất thế giới. Toàn bộ phase cloud (section 13-15, 24-25) của khoá này dùng AWS. Bài này không chỉ "click sign up" — ta sẽ học **cách setup AWS account đúng từ ngày đầu** để tránh **bị hack mất hàng nghìn USD** (chuyện có thật, xảy ra với nhiều người mới).

## Ba thực tế phải biết trước khi tạo tài khoản AWS

1. **AWS tính tiền theo giờ, theo byte, theo request**. Một tài khoản bị hack có thể bị spawn 100 EC2 instance mining crypto → bill chục nghìn USD trong vài giờ.
2. **Free tier có giới hạn**. Một vài service "free" chỉ cho 12 tháng đầu hoặc tới ngưỡng (vd 750 giờ EC2 t2.micro/tháng). Vượt = trả tiền.
3. **Root user** (email đăng ký) có **toàn quyền** trên account, kể cả đóng tài khoản. **Đừng dùng root user hằng ngày**.

3 luật trên dẫn đến 3 nguyên tắc trong bài này:

```text
1. Bật MFA root + IAM ngay
2. Tạo IAM user, dùng nó cho mọi việc — không dùng root
3. Set billing alarm trước khi tạo bất kỳ resource nào
```

## Bước 1 — Đăng ký AWS

1. Vào **aws.amazon.com** → "Create an AWS Account".
2. Email — **dùng email cho riêng AWS** (không phải email cá nhân). Tạo subaddress: `name+aws@gmail.com`.
3. Account name — tên dễ nhớ, không phải bắt buộc.
4. Verify email → chọn password mạnh.
5. Chọn loại tài khoản: **Personal** (không phải Business — đỡ thủ tục).
6. Nhập credit/debit card (verification, thường charge $1 sẽ refund).
7. SMS/voice verify phone.
8. Chọn support plan: **Basic (free)**.

### Free tier vs Paid tier

| Free tier (12 tháng) | Paid (Pay-as-you-go) |
|---|---|
| 750 giờ/tháng EC2 t2.micro hoặc t3.micro | Bất kỳ instance type nào |
| 5 GB S3 storage | Unlimited |
| 750 giờ RDS db.t3.micro | Bất kỳ |
| 1 triệu Lambda invocation/tháng | Free tier + over |
| 100 GB outbound transfer | Free tier + over |

> Đa số học AWS theo khoá sẽ **vượt free tier** ở vài resource. Đó là lý do bài 5 set billing alarm.

### Region — chọn ở đâu?

AWS có ~30 region toàn cầu. Mỗi region là cụm data center độc lập. Chọn region đúng quan trọng:

| Region | Code | Đặc điểm |
|---|---|---|
| **N. Virginia** | us-east-1 | **Cũ nhất, rẻ nhất, nhiều service nhất**. Mặc định khoá này. |
| Ohio | us-east-2 | Backup cho us-east-1 |
| Oregon | us-west-2 | Cheap, lots of features |
| Frankfurt | eu-central-1 | EU compliance |
| Singapore | ap-southeast-1 | Latency thấp cho Việt Nam |
| Tokyo | ap-northeast-1 | Latency tốt cho VN/Châu Á |
| Mumbai | ap-south-1 | Asia Pacific |

**Khoá học**: dùng **us-east-1**. Free tier ổn, mọi service mới release ở đây đầu tiên.

**Production thật**: chọn region gần user. Nếu user ở VN, có thể dùng `ap-southeast-1` (Singapore) — latency ~50ms thay vì 200ms từ us-east-1.

## Bước 2 — Bật MFA cho Root User

**ĐÂY LÀ VIỆC ĐẦU TIÊN BẠN PHẢI LÀM SAU KHI LOGIN**.

1. Đăng nhập console với email + password vừa đăng ký.
2. Top-right → tên tài khoản → **Security credentials**.
3. Section "Multi-factor authentication (MFA)" → "Assign MFA device".
4. Chọn **Authenticator app** (Google Authenticator / Authy).
5. Scan QR, nhập 2 mã consecutive 6 số.
6. Done.

**Hệ quả không bật MFA**:
- Nếu password bị brute force / phishing → hacker login được → spawn $10k EC2 ngay.
- Bot scan public GitHub commit có AWS keys → dùng root credential → tương tự.

## Bước 3 — Tạo IAM User (cho dùng hàng ngày)

**IAM** (Identity and Access Management) là service quản lý user, permission trong AWS account.

### Khái niệm cốt lõi

| Term | Ý nghĩa |
|---|---|
| **Root user** | "Owner" của account, có toàn quyền vĩnh viễn, không thể giảm quyền. **Chỉ dùng cho việc đặc biệt** (billing, đóng account). |
| **IAM user** | User do bạn tạo, có credential riêng, permission tùy chỉnh. Dùng hàng ngày. |
| **IAM group** | Tập user có cùng permission (vd Developers, Admins). |
| **IAM role** | "Tạm gán" permission cho user/service khác. Dùng cho EC2, Lambda, cross-account. |
| **Policy** | Tài liệu JSON khai báo "ai làm gì với resource nào". |
| **Permission** | Quyền cụ thể (vd `s3:GetObject`). |

### Tạo IAM user

1. Console → search "IAM" → IAM service.
2. Left sidebar → **Users** → "Create user".
3. User name: `devops-admin` (hoặc tên bạn thích).
4. ✓ "Provide user access to the AWS Management Console".
5. Console password: chọn "**Autogenerated**" → AWS sẽ tạo password ngẫu nhiên.
6. ✓ "Users must create a new password at next sign-in".
7. Next.
8. Permissions → "Attach policies directly" → search **AdministratorAccess** → tick.
9. Next → Create user.
10. **Quan trọng**: copy URL login + username + password (tạm). Hiển thị 1 lần.

### Account alias — URL đẹp

AWS cho URL login dạng `https://<account-id>.signin.aws.amazon.com/console`. Account ID dài và khó nhớ. Đổi alias:

1. IAM dashboard → "Account alias" → Create.
2. Đặt alias (vd `acme-learn`) — alias unique toàn AWS.
3. URL mới: `https://acme-learn.signin.aws.amazon.com/console`.

### Bật MFA cho IAM User

Sau khi user reset password lần đầu, login → IAM → Users → click user → Security credentials → "Assign MFA device".

**Mọi user có console access đều phải có MFA**. Đây là policy gần như bắt buộc trong mọi production org.

## Bước 4 — Access Key cho IAM User (cho AWS CLI)

Để dùng AWS CLI từ máy local, cần Access Key:

1. IAM → Users → click user → "Security credentials" tab.
2. Section "Access keys" → "Create access key".
3. Chọn use case: "Command Line Interface (CLI)".
4. Confirm.
5. Copy **Access Key ID** + **Secret Access Key** → lưu password manager.

Trên máy:

```bash
aws configure
# AWS Access Key ID: AKIA...
# AWS Secret Access Key: ...
# Default region name: us-east-1
# Default output format: json
```

File lưu ở `~/.aws/credentials` và `~/.aws/config`.

### Test

```bash
aws sts get-caller-identity
# {
#   "UserId": "AIDAxxx...",
#   "Account": "123456789012",
#   "Arn": "arn:aws:iam::123456789012:user/devops-admin"
# }
```

Nếu output ra đúng Arn → CLI hoạt động.

### Bảo mật Access Key

| Việc nên | Việc đừng |
|---|---|
| Lưu trong password manager / AWS Secrets Manager | Commit vào git |
| Rotate 90 ngày | Để 5 năm không đổi |
| Mỗi máy 1 key riêng | Share 1 key cho nhiều người |
| Có dùng MFA gắn theo (CLI session token) | Để Access Key đại trà |
| Set `.gitignore` cho `.env`, `.aws/` | Push lên public repo |

### Nếu lỡ commit Access Key

1. **Vô hiệu hoá key ngay**: IAM → Users → Security credentials → key → "Make inactive" → "Delete".
2. Tạo key mới.
3. Re-configure CLI.
4. Check CloudTrail xem có hành vi bất thường không.
5. Filter git history bằng `git filter-repo` hoặc BFG để xoá khỏi history (nếu repo còn private).

**AWS tự scan GitHub public**, phát hiện key sẽ tự khoá tài khoản tạm thời — nhưng đừng dựa vào.

## IAM Policy — cách permission thực sự hoạt động

`AdministratorAccess` là policy có sẵn cho phép **mọi action trên mọi resource**. JSON của nó:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "*",
            "Resource": "*"
        }
    ]
}
```

Trong production, không ai nên có `AdministratorAccess`. Thay vào đó dùng **least privilege**:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowS3ReadOnPaymentBucket",
            "Effect": "Allow",
            "Action": ["s3:GetObject", "s3:ListBucket"],
            "Resource": [
                "arn:aws:s3:::payment-prod-data",
                "arn:aws:s3:::payment-prod-data/*"
            ]
        }
    ]
}
```

Trong khoá học, vì là lab, dùng `AdministratorAccess` cho tiện. Khi vào team production, sẽ học viết policy nhỏ gọn (section AWS Part 2).

## "Defense in Depth" cho AWS account

Bảy lớp bảo vệ:

1. **MFA cho root + IAM users**.
2. **Không dùng root cho daily work**.
3. **IAM với least privilege** (production).
4. **Rotate access key 90 ngày**.
5. **CloudTrail** bật toàn account — log mọi API call.
6. **GuardDuty** — threat detection.
7. **Billing alarm** (bài tiếp theo).

Lab khoá học mới có #1, #2, #7. Production thêm phần còn lại.

## Bẫy mất tiền

| Bẫy | Cách mất tiền | Phòng |
|---|---|---|
| Quên tắt EC2 t3.large sau lab | $60/tháng | `aws ec2 terminate-instances` hoặc tag `auto-shutdown` |
| EBS volume orphan (sau khi terminate EC2) | $0.10/GB/tháng | Khi terminate, tick "delete volume on termination" |
| NAT Gateway không dùng | $32/tháng + traffic | Xoá khi không cần |
| Elastic IP không attach | $3.6/tháng | Release khi free |
| RDS Multi-AZ cho lab | 2x giá | Single-AZ là đủ cho lab |
| Snapshot cũ chồng chất | $0.05/GB/tháng | Lifecycle policy auto-delete > 7 ngày |
| CloudWatch log retention vô hạn | $0.5/GB/tháng | Set retention 7 ngày cho lab |
| Lambda + S3 vô hạn loop | Triệu invocation | Dùng dead-letter queue |

Một trong các lý do bài 5 dùng billing alarm: bắt sớm trước khi bill leo lên.

## Production scale — Organization và Multi-Account

Doanh nghiệp lớn không dùng 1 AWS account. Họ dùng **AWS Organizations**:

```text
Master / Management Account
   ├── OU: Production
   │     ├── Account: prod-app
   │     ├── Account: prod-data
   │     └── Account: prod-logging
   ├── OU: Non-Production
   │     ├── Account: dev
   │     ├── Account: staging
   │     └── Account: qa
   └── OU: Security
         ├── Account: security
         └── Account: audit
```

Mỗi account isolated → blast radius nhỏ khi bị hack 1 account. Section AWS Part 2 sẽ học sâu.

Trong khoá này: **1 account đủ**.

## Tóm tắt bài 4

- Sign up AWS với email/credit card → chọn Free tier + Personal plan.
- **Bật MFA root ngay** — bảo vệ đầu tiên.
- **Tạo IAM user** với `AdministratorAccess` cho daily work. **Đừng dùng root**.
- **Bật MFA IAM** — không có MFA, không touch resource.
- **Access Key** chỉ cho CLI, lưu password manager, rotate 90 ngày.
- Account alias = URL login dễ nhớ.
- 7 lớp bảo vệ — lab dùng 1, 2, 7. Production thêm 3-6.
