# Bài 1: Elastic Cloud overview

Phase 8 đã làm production ops manual. Tốn effort. **Elastic Cloud** = managed service (Elastic.co official) — họ manage infra, bạn chỉ dùng.

## Managed ES options

| Service                          | Vendor              | Notes                              |
|----------------------------------|---------------------|------------------------------------|
| **Elastic Cloud**                | Elastic.co (official) | All features paid                 |
| **Amazon OpenSearch Service**    | AWS                 | Fork OpenSearch, ES 7.10 max      |
| **Bonsai**                       | 3rd party           | Small/medium, simple              |
| **Aiven for Elasticsearch**      | 3rd party           | Multi-cloud                        |
| **Self-host on K8s** (ECK)       | DIY                 | Elastic Cloud on Kubernetes operator |

→ Elastic Cloud = canonical. Latest version, full features.

## Vì sao managed?

Self-host pros:
- Full control.
- Cheap (chỉ infra cost).
- No vendor lock.

Self-host cons:
- **Ops overhead** — patch, upgrade, monitor, scale, backup.
- **On-call** = team chịu trận.
- **Security** = bạn lo cert, encryption, audit.
- **Skill** — cần Elasticsearch expert.

Managed pros:
- **Zero ops** — vendor handle infra, patch, upgrade.
- **Up-to-date** — latest ES version always.
- **Built-in features** — security, monitoring, ML, snapshot.
- **SLA** — guaranteed uptime với credit refund.

Managed cons:
- **Cost** — 2-3× hơn self-host pure infra.
- **Lock-in** — migrate khó.
- **Limited customization** — không thể tùy biến deep.

→ Trade-off: time vs money. Startup/small team thường chọn managed. Big enterprise có DevOps team mạnh self-host.

## Elastic Cloud setup

### Sign up

1. <https://cloud.elastic.co/registration>.
2. Email + password.
3. **14-day trial** free, no credit card.

### Create deployment

1. **Create deployment**.
2. Choose:
   - **Cloud provider**: AWS / GCP / Azure.
   - **Region**: closest to user.
   - **Hardware profile**: General Purpose / Storage Optimized / Compute Optimized.
   - **Version**: latest hoặc specific.
3. Click **Create**.

Wait ~5 min. Deployment ready:

```text
Deployment URL: https://my-cluster.es.us-east-1.aws.found.io:9243
Kibana URL:     https://my-cluster.kb.us-east-1.aws.found.io:9243
Username:       elastic
Password:       <auto-generated, save now>
```

→ Lưu credentials password manager.

## Architecture

Elastic Cloud deploy:

```text
┌─────────────────────────────────────────────────────────┐
│  Deployment "my-prod-cluster"                            │
│                                                          │
│  ┌────────────────────────────────────────────────┐    │
│  │  Elasticsearch (3 zones for HA)                 │    │
│  │  ├ Hot nodes:     2× 8GB RAM × 3 AZ = 6 nodes   │    │
│  │  ├ Warm nodes:    1× 4GB × 3 AZ = 3 nodes       │    │
│  │  └ Cold nodes:    1× 2GB × 3 AZ = 3 nodes       │    │
│  └────────────────────────────────────────────────┘    │
│                                                          │
│  ┌────────────────────────────────────────────────┐    │
│  │  Kibana                                          │    │
│  │  └ 2× 1GB RAM × 2 AZ                            │    │
│  └────────────────────────────────────────────────┘    │
│                                                          │
│  ┌──────────────┐  ┌──────────────┐                    │
│  │ Integrations │  │ APM Server   │                    │
│  │ (Fleet)      │  │              │                    │
│  └──────────────┘  └──────────────┘                    │
│                                                          │
│  Snapshot: Auto daily → Elastic-managed S3              │
└─────────────────────────────────────────────────────────┘
```

→ Multi-AZ HA preset. Auto snapshot. Pre-configured ILM tiers.

## Pricing

Pay-as-you-go theo:
- **RAM** (primary cost).
- **Storage** (SSD vs HDD pricing differ).
- **Data transfer** (cross-AZ free, egress charged).

Examples:

| Profile               | RAM   | Storage | Cost/month |
|-----------------------|-------|---------|------------|
| Smallest (dev)        | 1 GB  | 30 GB   | ~$15       |
| Small prod            | 8 GB  | 240 GB  | ~$150      |
| Medium prod (HA)      | 32 GB | 1 TB    | ~$500-700  |
| Large prod (multi-tier) | 128 GB+ | 5 TB+ | $2000+     |

→ Self-host equivalent ~30-50% cost. Trade vs ops effort.

## Features included

Elastic Cloud bao gồm **Platinum tier** features:

- **Security**: SSO (SAML, OIDC), LDAP, RBAC, field/document-level security.
- **Alerting**: built-in.
- **Machine Learning**: anomaly detection, classification.
- **Cross-cluster search**: query multiple clusters.
- **Searchable snapshots**: frozen tier.
- **Reporting**: PDF export dashboards.
- **Canvas**: pixel-perfect reports.
- **Logs/Metrics UI**: Observability suite.
- **APM Server**: application performance monitoring.

→ Self-host basic = chỉ core ES. Cloud = enterprise full.

## Use cases

### Right fit cho Elastic Cloud

- Startup, no dedicated DevOps.
- Need quick POC / pilot.
- Want enterprise features without paying separate Platinum license.
- Compliance: official vendor, SOC2/HIPAA certified.

### Not great fit

- Air-gapped (govt, defense).
- Cost-sensitive at huge scale (PB).
- Need specific customization (custom plugin, kernel tuning).
- Already have Kubernetes platform → ECK self-managed maybe better.

## ECK (Elastic Cloud on Kubernetes)

Hybrid: self-host nhưng dùng **operator pattern** Elastic.co cung cấp.

```text
ECK = Kubernetes operator
        │
        ▼
   manages ES, Kibana, Logstash, Beats Pods
        │
        ▼
   auto rolling restart, scale, recovery
```

→ K8s yaml define cluster:

```yaml
apiVersion: elasticsearch.k8s.elastic.co/v1
kind: Elasticsearch
metadata:
    name: my-cluster
spec:
    version: 8.13.0
    nodeSets:
      - name: master
        count: 3
        config:
          node.roles: ["master"]
      - name: data
        count: 6
        config:
          node.roles: ["data_hot", "data_warm"]
```

`kubectl apply` → operator setup cluster.

→ Pattern modern cho team đã có K8s. Get ops automation + control + cost-effective.

## Cross-cluster search

Multi-cluster (vd geographically distributed):

```text
Cluster EU ──┐
              ├──► Search both
Cluster US ──┘
```

ES feature: register remote cluster, query as if same index:

```text
GET /eu:logs-*,us:logs-*/_search
{
    "query": { ... }
}
```

→ Logs từ EU + US merge. Disaster recovery + global view.

Setup:

```text
PUT /_cluster/settings
{
    "persistent": {
        "cluster.remote.us": {
            "seeds": ["us-cluster-host:9300"]
        }
    }
}
```

## Migration self-host → Cloud (hoặc ngược)

Snapshot pattern:

1. Snapshot self-host cluster → S3.
2. Register S3 repo on Elastic Cloud.
3. Restore.

→ Data move seamless. Mapping, settings preserve.

App connection string change → done.

## Tóm tắt

- **Elastic Cloud** = official managed service. AWS/GCP/Azure backend.
- Trade-off: cost 2-3× vs self-host, nhưng zero ops + enterprise features.
- 14-day trial free.
- Multi-AZ HA preset, auto snapshot, ILM tiers configured.
- Platinum features included: security, ML, alerting, APM, reporting.
- **ECK** = Elastic operator on Kubernetes — hybrid self-host + automation.
- **Cross-cluster search** cho multi-region.
- Migration via snapshot to S3 + restore.
