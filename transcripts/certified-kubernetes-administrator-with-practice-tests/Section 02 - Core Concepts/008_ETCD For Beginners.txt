So relational databases that we just discussed have a strict schema, can perform complex queries using SQL delivers good performance however, is very rigid in terms of flexibility and is best for structured data. A document store. Stores information in the form of documents, so each individual gets a document, and all information about that individual is stored within that file. These files can be in any format or structure, and changes to one file does not affect the others. The working individuals can have their files with salary fields, and the students can have their pages with grades only. You can add additional details to any of these documents without having to update other similar documents. Document stores typically store data in a JSON format, so comparing relational databases to document store document stores typically don't need a schema defined can be limiting in terms of performing complex queries by joining different tables. Delivers good performance, is very flexible thanks to the document structure, and is best suited for semi-structured data, as we call it now. Key value store does not necessarily have a schema, does not support complex queries.
