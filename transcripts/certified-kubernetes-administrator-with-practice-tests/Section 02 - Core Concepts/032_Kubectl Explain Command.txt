# Kubectl Explain Command

Let us now look at kubectl explain command. So there is a lot you can learn about resources and fields without ever leaving the terminal. To list all resources you can simply run kubectl api resources command. This is especially useful when you can't find a resource on the documentation pages. So if you ever forget a resource name or its short name, or you're not sure of the underlying version of the API such as V1 or is it v1? Or if you are not sure the exact case of the resource name to be used while creating a definition file, you can use the kubectl API resource command. Once you identify the resource name from the list and you need more information about the resource fields and types under it, you can then run the kubectl explain command. Provide the resource name that you'd like to be explained in this case, using pods. Lists all top level fields in pods being the API version kind, metadata, spec and status, and you can see the type of each field. So the API version is string and also the description of what it does. But as you can see, this is not a comprehensive list of all fields and subfields. It only lists the top level items. To go deeper, run the same command but with a field to the command. And like this. So you would add dot spec to the end of the command and this would list the subfields within it. But this again lists the subproperties of parent property that you've mentioned. It's still not comprehensive. So to list all fields in the way that you would put them in a YAML file, use the recursive flag like this. It now outputs the entire comprehensive list of fields that's available on that resource. And it's very easy now to create YAML files out of this.
