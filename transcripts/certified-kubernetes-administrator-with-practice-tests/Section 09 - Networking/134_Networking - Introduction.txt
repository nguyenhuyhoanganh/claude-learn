# Networking - Introduction

-: Hello and welcome to this section on networking in Kubernetes. My name is Munjab Manamet, and we are going through the certified Kubernetes Administrators course. In this section, we focus on the networking concepts. Some networking basics are required to get through the section, such as the basics about setting and checking IP addresses on any given system, knowledge on gateways and routes, and knowledge about DNS and configuring DNS servers, et cetera. While working on this section, as before, we sent out a poll asking about your knowledge in networking. Based on your feedback, we've created some prerequisite lectures around configuring interfaces and IP addresses, configuring gateways and routes, basics of name resolution, DNS configuration on Linux systems, as well as how to get started with one of the DNS solutions core DNS. These are short lectures that should give you enough knowledge to go through the remainder of this section. It is also important to know a little bit about a network name spaces and how Docker uses network name spaces internally. Again, as always, these are for beginners and our optional lectures, and depending on your knowledge, feel free to go through only those that you think you really need. I would highly recommend the network name spaces and Docker networking lectures. Once we are done with the prerequisite lectures we start with networking in Kubernetes. Again, I'm going to rearrange the topics a little bit to improve the flow. We start by looking at the cluster networking needs and then followed by POD networking concepts. We then look at CNI in Kubernetes and how CNI can help solve the POD networking challenges. This is where we pick up from where we left off in the prerequisite lectures around network name spaces. We continue to discuss cluster DNS and how Kubernetes implements DNS internally. We then finish with ingress networking. Again, this is one section that I spent a lot of time on and I hope you enjoy it.
