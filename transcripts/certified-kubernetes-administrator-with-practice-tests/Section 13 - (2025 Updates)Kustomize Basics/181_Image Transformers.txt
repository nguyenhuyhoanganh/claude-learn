# Image Transformers

(lounge music) Narrator: Let's move on to the image transformer. The image transformer is what's going to allow us to modify an image that a specific deployment or container is going to use through customize. So in this example, I have a deployment.yaml file where we're going to deploy a nginx server. And the image right now is set to nginx, but we can transform that using customize. So let's define our customization.yaml file and we're gonna have to specify two properties. There's a name and then a newName. So the name is going to reference the name of the specific image that we want to replace. So this is going to look through all of our Kubernetes configs and it's going to find all the containers that use an image called nginx. And then we're going to replace it with the newName, which is haproxy, so it's going to replace that image of nginx with haproxy. Now just to kind of clear up some potential confusions, you may notice that under the customization.yaml file, we have a name, and then under the containers under the deployment.yaml file, we have a name as well. These two things are not related. So the name under the customization.yaml file specifies the name of the image, not necessarily the name of the container. So ignore the name of the container that's set to web, we don't care about that. The name property under customization.yaml file looks specifically for an image called nginx. Something to keep in mind, it confused me at first, so I figured you guys would probably run into the same issue. And so after we apply this, we can take a look at the new images now set to haproxy. So now we know how to change the image using customize. However, let's say that we don't actually wanna change the image, so we've got our nginx image here as well, but let's say we just wanna change the tag. Well, we can use the image transformer to perform that action as well. So instead of using the new image property, we can assign the new tag property. So this is going to give the new tag that we wanna assign to that image. So it's going to look for all of the containers using an nginx image, and then it's going to add a tag of 2.4. So if we take a look at the final result, we can now see it's nginx:2.4. Now if we want to, we can actually combine the new image and new tag properties together so that we can modify not only the image, but the tag. So if we wanna change our nginx image to be haproxy and assign a tag of 2.4, we can do that all within an image transformer and the final result is going to be haproxy:2.4.
