# Module Introduction

In the previous course section, we got started with the basics of GitHub Actions with the general syntax you use in those YAML files for defining your actions and with the core features that make up GitHub Actions. In this course section here we're going to dive deeper into GitHub Actions and we're going to start by diving deeper into the events that can trigger a Workflow. Because as it turns out, you cannot just add simple event definitions where you, for example say that pushing to a Repository should trigger a Workflow but you can set up more complex conditions, and for example also add event filters, so that, for example not all push events trigger a Workflow. You can work with different activity types and you will of course learn what activity types are and you will learn all these things and more and some concrete examples so that you don't just learn it in theory.
