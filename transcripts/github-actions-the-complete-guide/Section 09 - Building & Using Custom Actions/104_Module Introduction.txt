# Module Introduction

At this point in the course we worked quite a bit with GitHub Actions and with a lot of different features provided by GitHub Actions. And one feature which we used a lot already in our workflows are actions. Actions like the cache action here, or the checkout action or the action for uploading artifacts. We used such actions in our steps in our jobs to perform various tasks. And the idea behind such actions is that they can perform any task of your choice of any complexity, simply depending on what a specific action does. For example, upload artifact takes some files and makes them available outside of the workflow or in subsequent workflow jobs and their steps. So these actions simply perform all kinds of tasks and things in single steps and that's the idea for using such actions. But up to this point in the course we always used public existing actions created by the GitHub team or other community members. We did not create any custom actions ourselves. But that is also something you can do and that is what we'll do in this course section here. For that, we're first of all going to explore why we might wanna build custom actions and what exactly custom actions are, before we then dive into the different types of custom actions you can build because there's more than one kind of custom action you can build. And before we then of course also will build and use custom actions so that you also see all these things and concepts related to custom actions in action.
