# Beyond AWS

Now AWS was just an example in this section. But no matter which provider you choose you will often have this choice between using some services where you have full control, you have your own remote machine on which you install Docker, but where you'd offer also I have the full responsibility and a managed service which allows you to deploy containers. And I showed you both in this module, ATC example of AWS. We started with EC2, which was that installed it on your own approach, but then we moved to ECS to have this managed service. And especially that manual installation approach will be available by many providers because having a remote machine which you can manage on your own is fairly easy to sell. More complex managed services like ECS are also present on a lot of providers not all of them though, but especially the big providers of course all have similar Services. Now, I already mentioned it earlier in this course, I can't cover all providers and of course the exact steps, especially for the managed services differ between providers. My goal was to show you these two main philosophies. You could say, for deploying containers installing Docker on a remote machine where it's managed services, and I wanted to show you potential challenges what to keep in mind and how certain features we saw locally with Docker and Docker compose can be translated to manage services. Therefore what you learned here should not just help you with AWS, but with any provider since you already are aware of a couple of approaches and pin points, you might be using and facing when deploying containers.
