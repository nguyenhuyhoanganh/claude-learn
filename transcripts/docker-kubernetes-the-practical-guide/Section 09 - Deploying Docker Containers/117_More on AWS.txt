Now, I also briefly wanna explain this Fargate thing in greater detail. We did create our first cluster and run our first container in this task which uses this Fargate thing, this Fargate runtime, you could say. Now, this is AWS specific, and it simply means that AWS will run this container in an environment where it only creates this server on which the container runs when it's needed kind of, of course it does this in a very efficient and fast way, but it will ensure that we don't always have a EC2 Instance, as server up and running, but only when we need it. And that server is also fully managed for us, so that we don't need to worry about updates or anything like this. The task size which we can choose down there, then simply specifies how much memory and CPU is available for our container when it executes. And here we by default use the smallest possible choices, of course, if you are using this in production, you might wanna consider using more powerful settings here, but of course the more powerful settings you choose here the higher your costs will be. So, for the demo purposes here the smallest settings will do. Now of course, if you use the smallest possible settings here, this means that if you got a lot of incoming requests, your container will be up and running handling them. But at some point, these resources might not suffice. AWS, ECS then also offers something which is called Auto Scaling, where it would actually create more than one running container at the same time, with these tasks sizes to help you scale up and have more containers handling incoming requests.
