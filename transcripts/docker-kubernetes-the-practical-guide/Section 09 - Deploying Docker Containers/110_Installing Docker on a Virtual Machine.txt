# Installing Docker on a Virtual Machine

So let's start with installing Docker. For this here in your connected terminal where we are running commands on the remote machine, start with sudo yum update dash y. This will simply ensure that all essential packages on that remote machine are updated and are using their latest version. Then after, run sudo amozon dash linux dash extras, install Docker. Now that's why we should pick this Amazon image when we create it. The "ec2" instance. On these Amazon based virtual instances we have this command available which makes installing extra software like Docker, super easy. And this will now install Docker on that remote machine. We installed it on our local machine in the first core section, but that won't help us if we want to run a container on the remote machine. That's why we need to install Docker there as well. And that's what this command does. It installs Docker on this remote machine which is running in the cloud. Let's confirm this with y. And now it installs all the required tools and Docker itself, which are needed to run Docker there. So let's wait for this to finish. And with this finished, we can run sudo service docker start to start Docker there. And then after once Docker is started, we are able to run these Docker commands like docker run. If I now hit enter you see it complains that it needs at least one argument but it does not complain that it wouldn't know Docker. So that proves that Docker is now available here.
