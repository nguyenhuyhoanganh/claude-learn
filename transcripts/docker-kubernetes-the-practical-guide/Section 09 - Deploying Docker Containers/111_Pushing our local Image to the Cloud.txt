# Pushing our local Image to the Cloud

So now that Docker is installed and up and running on that remote machine, we need to bring our local image onto that remote machine. And for this, we got two main approaches. Option number one, is that we deploy our source code. So, we find a way of copying our source code, so everything in this project folder, including the Dockerfile file onto that remote machine and then we build it there, we build the image there. The alternative is that we build the image ahead of time, for example, on our local machine. And we then just deploy that built image onto the remote machine. So, it's simply about building the image on the remote machine and doing that by pushing our source code there and running Docker build there and then Docker run, or if we want to build it locally, for example. And then just execute Docker run on the remote machine once we got the image there. And the first option has a lot of unnecessary complexity to it, to be honest. We need to do a lot of unnecessary work on the remote machine. Why would we build the image there? Which advantage does this give us? I'm not sure if it gives us any advantage. I'd rather prepare everything locally and have the finished image, locally, and then just get my finished image onto that remote host to then run it there as a container, and we can use Docker Hub for that. We can build our image locally, then push it to Docker Hub and then pull it from the remote server to then run it on the remote server. And that's exactly what we're going to do.
