# Docker Compose

Speaking of multi container projects, we also learned about Docker Compose. The idea behind Docker Compose is simple. With Docker, we can run containers and build images, but the commands for doing that might get rather along. Especially if we have multiple environment variables, volumes, or networks. Then we have very long Docker run commands, and especially in multi container environments, that then quickly becomes a lot of work. Therefore, we have Docker Compose. Docker Compose allows us to put our container launch configuration, into a configuration file, which is sent automatically picked up when we run Docker compose up, which then starts and initializes, all the containers as defined in that configuration file. And it's all the den, just whenever a command, docker composed down, to bring all these started containers, down again, if we wanna stop them. Therefore, Docker compose is an amazing tool, for more complex containers and more complex projects, especially, but not limited to, multi container projects, Of course.
