But since with Kubernetes we run our application possibly across multiple nodes, and possibly on different cloud and hosting providers. So the volume lifetime, because the volume is part of the pod, depends on the pod lifetime. With Kubernetes volumes, you have different drivers and types. Which gives you great control over where the data is stored. Now, when you work with just Docker and Docker compose, that's not the case, because that's also not something you really need. So when you use just Docker and Docker compose you don't necessarily need that much support for different storage systems. But with Kubernetes of course, running your application on a cluster with multiple nodes, on different hosting environments.
