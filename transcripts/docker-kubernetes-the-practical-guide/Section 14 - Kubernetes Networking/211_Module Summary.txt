# Module Summary

And that is it for this module. We had a look at all the important scenarios when it comes to containers and pods interacting with the Java and the outside world. We had a look at pod internal communication through local host. This special address which you can use if two containers are running in the same pod and one container wants to send a request to the other container. We had a look at outside world communication with type load balancer. So, by creating such a service of type load balancer. We had another look at services in general but we also had a look at cluster internal communication. When we want to ensure that we actually have multiple pods talk to each other and maybe some pod is also only reachable from inside the cluster. We learned that we can achieve this by creating a service of type cluster IP which also has some automatic load balancing built in as I mentioned and TB10 can either look up the IP address manually or that we use this automatically generated cluster internal domain name or that we use the automatically generated environment variables which Kubernetes gives us. And this allows us to really enable exactly the setup and the communication we need in our deployed application. We can lock down what we wanna lock down and for example, ensure that the Auth API simply can't be reached from outside and we can still implement all the communication we need with all the tools I showed you throughout this module.
