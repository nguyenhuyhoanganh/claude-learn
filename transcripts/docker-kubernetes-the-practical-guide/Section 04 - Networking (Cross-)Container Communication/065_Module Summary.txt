# Module Summary

Now this is already it for this module which was about container communication and networks. Now there is more you could do to optimize this application and improve how containers are started. For example, for the moment we always lose all the database data whenever we stop the mongoDB database. This is something we could solve with volumes and it actually is something we will solve with volumes. But we'll take a look at that, combined with a couple of other things in the next module when we will dive deeper into building multi-container applications because most applications are multi-container applications. So we'll take a close look at that in the next module. For this module it's important that you understand that requests to web servers and so on are always possible in containers. Requests to the local host machines are easy to do with that host.docker.internal domain here and that cross container requests are easy if you look up the IP or even better if you create a network and then you just use the container name and let docker do the handwork . That's what you should take away from this module.
