For that, we first of all need a second container, because up to this point, we always had only 1 container working on its own. Now I want to have 1 container for the note app and one container for the MongoDB database, Because as I mentioned earlier, already, it is a good practice to have every container focus on just one thing. For that, let's start by creating a MongoDB container and you could think that we're going to write another Docker file for that, but we don't even need to do this, because if you search for Docker MongoDB, you will find an official Mongo image on Docker Hub. So just as before, with the official node and Python images, this is an official MongoDB image. Now you'll find detailed instructions on how to use it on this Docker Hub page. That's true for all kinds of images by the way. But we're going to set it up step by step together anyways, so you don't need to go through these instructions for now. Instead, what we're going to do here is we're going to run a container with Docker run based on this Mongo image. Now that is an image which already exists here on Docker Hub. So I'm using this official image named Mongo. If I run Docker run Mongo, this would create a new container based on this MongoDB image and this image will spin up a MongoDB database. So if I hit Enter here, you should see some output like this and this is now the running MongoDB database container. Now, I want to alter my node js application code such that we can connect to this database running in this upper container and that's now the tricky part, host.docker internal will no longer work, because that refers to my local host machine IP address, not to the IP address of some other container but instead we can do something interesting, we can run a number of command and that's the Docker container inspect command to inspect this MongoDB container we just started. Here we get a lot of information about this container, but the interesting bit of information can be found if you scroll up a bit, this IP address here, which you find under network settings in this object here, there this IP address. It's a totally separate, isolated database. After all, separation and isolation are core concepts and core reasons for using Docker, right. So we got no data back because there initially is no data but and that's super important. But of course, it's not very convenient. We had to look up the IP address of the our container in order to then use it here. Definitely not a lot of work, but still quite cumbersome. It also means that we always have to build a new image whenever the MongoDB container IP address changed, because we hard code that IP address here ,in the node app and this of course, is not ideal. So thankfully, there is an easier way of having multiple Docker containers talk to each other.
