# You Learned A Lot!

So that's it for this course and for Docker Kubernetes, Docker Compose and everything we covered over the last course section. This was definitely no short course. A lot of content was covered, and you definitely learned a lot about Docker and how to use it locally with Docker Compose, how to deploy it and how to work with Kubernetes. You got a very solid foundation throughout the course. You learned what exactly Docker is, what containers are and why we might wanna use Docker and containers and how we use it of course. To be precise, you learned how to work with Docker locally on your local machine, how to build images and containers, how to start them, how to configure them, how to have multiple containers work together, and you also learned how to deploy containers either manually, by fully managing your servers, by using a managed service like AWS ECS, or of course with help of Kubernetes. We had a very close look at images and containers, the two core concepts that make up Docker. And we had a look at how to configure and build images, how to use existing images and how to then also run these images as containers. And not just like this, but also how to let these containers communicate with the outside world or other containers and we learned how to use a volumes and much, much more. We also learned about Docker Compose, which is a tool that makes managing multi container projects easier, or that could even be used with just one container, because you can encode the entire runtime configuration for that container in one convenient configuration file, which is easy to share and adjust and which saves you the hassle of typing long Docker Run commands all the time. Docker Compose offer is an awesome tool, especially locally on your local machine. If you wanna spin up your Dockerized projects with ease. But then we also had a look at Kubernetes which is kind of like Docker Compose for more complex, multi node deployments. There you learn how to work with Kubernetes how to set up a Kubernetes cluster and how to then orchestrate and deploy your multi container application with Kubernetes. You learned about the important internals of Kubernetes about pods and containers, how to launch and restart and scale pods and also how to let pods and containers communicate with each other and how to add volumes. And with all that, you got a very solid set of fundamentals, actually, even more than just that and you're well prepared to use Docker and containers in your next project, whatever that is. You can use them just locally, just in production or both because in this course you learned all about containers you need to know for that.
