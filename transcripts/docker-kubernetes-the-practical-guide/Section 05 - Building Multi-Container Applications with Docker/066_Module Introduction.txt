# Module Introduction

In the last module, which was about networking, We started building an application that used more than one container. In this course module, we are going to continue on that road and we're going to build a multi container application and therefore a more realistic kind of application. Because whilst you could absolutely also have applications that really only need one container, more on that in the deployment section later, you often will have more than one container. You often will have applications that consist of multiple different services working together. For example, a database, a backend web server, and a front end application, something like this. Now the idea behind this module is to show you how you can build a more realistic application consisting of multiple services and therefore multiple containers which work together. But the idea is also that you can practice what you learned over the last course modules. Because we are not really going to learn anything new in this section, rather, we're going to combine what we learned and apply what we learned to this multi-service, multi container application. Specifically in this module, we are going to have a look at how we can combine multiple building blocks, multiple services to one big app, and how we can use Docker for that to work with multiple containers. Now I will show you which kind of app we're going to build, or we're going to Dockerize in the next lecture, but thereafter, definitely also feel free to pause and to try this on your own first, before you continue together with me, because this will also be a great practice for you to apply what you learned over the last sections in a more realistic application.
