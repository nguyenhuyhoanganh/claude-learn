# Module Summary

So that's it for this module already. I introduced you to Utility Containers. Again, that's not an official term, but I think it's describes quite well what these containers are about. They don't include an app, which is started. Instead, they just have an environment, which you can use to run certain commands. And you can use this with docker run. But it's even more convenient with docker compose run, because then you can put all the configuration you might need, for example, a bind mount, which you will use quite frequently with such Utility Containers into this configuration file. And then you have one very simple command to execute any kind of command inside of your container here. And that is a pattern which can be quite useful, because it means that you can set certain things up on your local system without installing all the tooling first. And a really great example for that will be a Laravel application. Laravel is a PHP framework, the most popular PHP framework, and setting up the development environment for it can be quite challenging. Well, not so much with docker. So in the next section, we'll have a little practice section, where we will utilize what we learned about containers and compose and also Utility Containers to create a dummy Laravel application.
