# Adapter - Summary

Now we're going to discuss summary of adapter design pattern in this video. So whenever we have an existing object that provides the required functionality required by some part of code, but that particular code is expecting a different interface than what our object has implemented. In that case, we can use our adapter design pattern. So there are two ways you can implement this design pattern. The first one is called as a class adapter. In class adapter implementation, we inherit from our existing class whose object we already have, and we implement the target interface which is expected by the client code. This way we have a class adapter. Now this type of adapter should be avoided. Second way to implement our adapter is called as an object adapter and this adapter uses composition. So instead of extending from our existing class, this adapter will simply implement the interface which is expected by client code, and it will use the adapter object, the object that we already have as in a composition in order to perform the operation which is requested by client. Now, one benefit of using object adapter is that we can substitute the object that is being used by adapter by one of its subclasses. Objects of one of its subclasses.
