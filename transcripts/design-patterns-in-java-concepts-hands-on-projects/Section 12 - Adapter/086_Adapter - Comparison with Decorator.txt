# Adapter - Comparison with Decorator

Now let's compare and contrast our adapter design pattern with another structural design pattern called as decorator. Now, we haven't yet discussed decorator, but it's quite a simple design pattern and in structure it is quite similar to how we implement object adapter. Okay, so in Decorator as well, we have a class which contains an object of our existing class. So what are the differences between adapter and decorator? Well, first of all, adapters purpose. The intent behind using adapter design pattern is to simply adapt an existing object to a different interface. And we do not intend to change the behavior that is provided by our existing object decorator. On the other hand, the purpose of decorator is that it enhances the behavior of our existing object, but it keeps the interface same. A decorator is indistinguishable from the object that it decorates. Both of these patterns use composition the adapter with object adapter, design pattern and the decorator. They both use object composition in order to achieve their functionality. Another difference between adapter and decorator is that using adapter we it is not easy to perform recursive composition, so a recursive composition is where we have one adapter which contains another adapter, which contains another actor and so on and so forth. So it is not easy to do that with adapter because they change the interface from the existing object. In decorator. However, a decorator is completely indistinguishable from the original object because it maintains or keeps the exact same interface as the original object. So that's why it is possible that we can have a decorator which contains another decorator because they are indistinguishable from the original object. They can. We can do recursive composition using decorator design pattern. All right. So these are differences between adapter and decorator design pattern.
