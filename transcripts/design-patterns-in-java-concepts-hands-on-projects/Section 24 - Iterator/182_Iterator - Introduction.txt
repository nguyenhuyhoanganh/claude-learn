So whenever we have an aggregate object, an aggregate object is an object which has other objects as its state. So it has a collection inside it. And if we want to give access to these elements which are inside our aggregate object in a sequential manner, then we can use iterator. Using iterator allows you to hide the actual data structure that is used in our aggregate object. So our aggregate object might have an array inside it, or it might be using any inbuilt Java collection. And this object will provide a method to get an iterator which can be used by our client code to iterate over its elements or children. Now in Java language iterators are integral part of the collection framework, and these iterators that you use in Java are actually implementation of this same design pattern that we are going to learn here. Now one thing you should note is that iterators are stateful. That means they remember where they are while we are iterating the internal collection so they remember the position. All right.
