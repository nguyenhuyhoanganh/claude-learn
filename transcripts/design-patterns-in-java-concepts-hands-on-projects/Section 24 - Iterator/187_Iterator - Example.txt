# Iterator - Example

All right. Now let's look at some examples of iterator design pattern in real world. And yes, the iterator classes that are present in Java collection frameworks are great examples of this design pattern. You cannot ask for a better example of any design pattern. The concrete iterators. One thing that I would like to highlight is that the concrete iterators are typically inner classes that are present in each individual collection class. For so for linked list and list, we will have some internal inner class that implements that iterator interface. There is another class in Java util package called as Scanner, which is also an example of iterator design pattern. One benefit of this or one speciality of this class is that this class supports input stream so we can iterate over incoming data from our incoming stream and get information from that stream using the iterator interface. So scanner is also an example of iterator design pattern. Then there are classes like XML Event Reader, which is part of Java API for XML, and these classes allow us to treat the XML into a stream of events and each event object is basically a structure or element or tag in that XML document. So XML event reader class is also an example of iterator design pattern.
