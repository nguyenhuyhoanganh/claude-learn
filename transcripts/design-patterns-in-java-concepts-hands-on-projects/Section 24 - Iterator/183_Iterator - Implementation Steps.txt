# Iterator - Implementation Steps

All right. Now let's look at typical steps that you would follow while implementing iterator design pattern. So we start by defining our iterator interface, and our client works exclusively with that interface. So this interface has to provide a couple of methods. First of all, it needs to tell the client whether there is an element available for consumption. So we have this Hasnext method that returns a Boolean to indicate if our iterator can provide next element and we have the actual method which returns the object and it is typically called as next method. Now we then implement the iterator and typically you will see that we have iterator implementations as inner classes because implementing them as inner classes allows us to give these iterator implementation access to our internal data structure. Because if they are not internal classes, then we would have to expose our internal data structure. But that is what we don't want to do in first place. That's why we have the iterator. All right. Having an inner class means that it can access the private or protected methods in our aggregate object or aggregate class. Now, concrete iterator needs to have some state to identify where it is in the underlying collection. Now, if that underlying collection changes, then an iterator should throw an exception to indicate the client that whatever state it has is now become invalid and the iteration may not be possible to continue. And it is the same scenario we discussed while introducing this design pattern that if somebody calls a hasnext method on our iterator and it returns true because there was an element available in the collection for consumption and in the meantime, if somebody cleared out that underlying collection and client called the next method, that means we have violated our contract because we just told someone that, okay, yeah, I can give you a new object. And when you actually asked for it, we don't have that object. So that means this is an exceptional scenario. All right. So this is how we can implement the iterator design pattern.
