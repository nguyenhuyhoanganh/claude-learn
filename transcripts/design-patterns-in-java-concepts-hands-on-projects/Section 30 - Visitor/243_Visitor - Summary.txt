All right, Now let's summarize visitor design pattern. So we use visitor when we want to provide new functionality or provide new operations that work on objects without modifying classes defining those objects. So visitors define class specific methods which work with an object of that class in order to provide a new functionality. Okay, so to use this pattern, classes define a simple accept method which gets a reference to our visitor object. So we pass a visitor object to that, accept method and inside the implementation of that accept method. Each class is going to call a method on that visitor object which is specific to that class. Adding a new functionality using this design pattern means creating a new visitor and implementing that new functionality in that class. So instead of modifying our existing classes, we are simply going to create a new class which implements our visitor interface and provide the functionality in that new class. Now, this pattern is often used where we have an object structure. So we could have a tree structure, we could have a list or any other collection. So whenever we have an object structure like that and we want to provide some functionality to all those objects, then this is where the visitor pattern is typically used. All right.
