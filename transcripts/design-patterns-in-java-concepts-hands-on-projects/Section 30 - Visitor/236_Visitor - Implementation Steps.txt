# Visitor - Implementation Steps

All right. Now let's look at a sequence of steps that we would typically follow if we are implementing visitor design pattern from the introduction video. You might have gotten a clear idea of what we are going to do. So we start by creating our visitor interface. Inside this interface, we are going to define multiple visit methods. And each visit method will be specific to a concrete class that our visitor supports. So if you want to support two concrete classes, we are going to have to visit methods. Each method will be specific to a concrete class. Now this visit method is going to expect a reference being passed to it, which is of that concrete class. Then we come to the objects or classes. We want to use that visitor functionality. So their only requirement is that they should provide an accept method where we can pass on a visitor reference. So we can create a interface and define our accept method. There and have our classes implement that interface. Or we can directly define the accept method on these objects. Now these methods are defined using visitor interface as a parameter type. So that means we can pass any subclass or any class which implements that visitor interface. Inside these accept methods in the implementation, we are simply going to call method on the visitor which is specific to the particular class. Next we implement the visitor interface in one or more classes, and each implementation will provide a specific functionality for all the interested classes. And if we want another feature, then we will create another class and then we will provide functionality there. All right, so this is how we can implement visitor design pattern.
