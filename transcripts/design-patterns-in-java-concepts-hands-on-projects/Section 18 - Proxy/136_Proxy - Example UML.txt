All right. Now let's look at the example of the example that we're going to implement. So what we are going to implement is called as a virtual proxy. So we are going to create a standing object for the real object, and we are going to postpone creating real object until possible. Okay. And let's look at the participants in this particular implementation. So first of all, we have an image interface. So this is the interface which is used by our client. And for this interface we have two implementation. We have a bitmap image class, which is the real object, which has the real data, and then we have image proxy. So this is our proxy class. So this is again going to implement the image interface as we discussed, and it's going to have a reference to the real object. Then we are going to provide an image factory which is used by client, and this factory is transparently create our instance of proxy and return to client. So client is not aware that he is dealing with a proxy object. It's going to work with the image interface only. All right. And when it comes time that we need a real object, for example, if the client calls the render method now we need the actual data or actual bytes which are part of that image. And then only we are going to create the real object, load it from the disk and then perform the rendering. Okay. We are not actually going to do a rendering, but we are going to provide a method that is a placeholder for that.
