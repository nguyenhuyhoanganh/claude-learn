# Proxy - Dynamic Proxy Implementation Steps

All right, Now let's see how we can implement a dynamic proxy in Java. So this is a feature that is provided by Java out of the box, and it allows you to create proxies at runtime. Now, we saw that the reason we create proxies is because we want to execute some code whenever a method is called on the actual object. So in order to use dynamic proxy, we create a class. But this class implements invocation handler interface, which is provided by Java. Now this interface has a single method called as invoke. Now this method is called for every method call that happens on our proxy instance. So that means inside this method we have to first check which method was called on the proxy and then decide what we want to do now. Okay. Our invocation handler will accept the same argument in the constructor as needed by constructor of real object for which we are creating proxy. So this is the invocation handler. What about the actual proxy? To create a proxy instance, we can use another class called as proxy, which is again provided by Java. So we're going to see how to use this invocation handler interface and the proxy class in order to create a dynamic proxy.
