# Proxy - Example

All right, Now let's look at some examples of proxy design pattern in real world applications. Now you will see that there are there is no problem finding is finding examples of proxy design pattern because it is used widely. So to name a few examples, Hibernate uses proxy design pattern whenever it is loading collections of value type. So if we have a relationship in our entity class that is mapped as a collection, then we can mark that relationship for lazy loading. So whenever a collection is marked for lazy loading or relationship is marked for lazy loading, Hibernate will create a virtual proxy and put that reference in that particular field. All right. Spring also uses proxy design pattern in order to provide support for features like transaction management, caching and general AOP support. Now, one point that I want to highlight is that both of these frameworks can create proxies for classes at runtime, and even if those classes do not implement any interface and they do that using third party frameworks like zlib and aspect to create those proxies. Now remember, Java's dynamic proxy needs our class to implement at least one interface so that we can create proxy dynamically at runtime. All right. So these are few examples of proxy design pattern, and I'm sure you will find many examples when working with web services. When working with RMI, we use proxies in a lot of different places.
