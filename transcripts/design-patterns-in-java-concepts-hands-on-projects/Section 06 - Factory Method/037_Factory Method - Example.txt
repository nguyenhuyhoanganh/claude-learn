# Factory Method - Example

All right. Now it's time to look at an example of factory method in real world. So the Java.util.collection interface or the abstract class, which is present in the same package, which is the abstract collection class, has a method called as iterator. Now this abstract method is an example of a factory method. So let's look at the code snippet from the abstract collection class. Remember, this code is taken from the Java Util dot abstract collection class from the jar file. In this class you will see that we have a method here called as iterator. And purpose of this method is that this method will return an object which implements the iterator interface which is used by the client code to iterate over the collection. All right, so this iterator method, which is present in the abstract collection, or you can say even the collection interface is an example of factory method. Now remember the most defining characteristic of factory method pattern is that subclasses are providing the actual instance. So you will see many examples online where people are giving static methods as an example of factory method. But these are not actually the factory method design pattern examples from Gang of four book. Because in a factory method design pattern, the base class defines the factory method, which is iterator method in this example. And subclasses of this class will actually provide a concrete implementation of that particular class. All right, so we have classes like ArrayList Linkedlist, which are subclasses of our abstract collection, and those classes will implement the iterator method and they will create an object in those overridden methods and then return those objects to the client code. All right. So the iterator method in the abstract collection is an example of factory method design pattern.
