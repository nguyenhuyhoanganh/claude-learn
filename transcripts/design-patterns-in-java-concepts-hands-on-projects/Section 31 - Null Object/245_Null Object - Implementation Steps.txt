# Null Object - Implementation Steps

All right. Now let's look at a typical sequence of steps that we would follow when implementing null object design pattern. So we'll start by creating a new class that represents our null object. And we write this class by extending from a base class or implementing a given interface. Now in the null object implementation. So in this new class for each method, we will not do anything. So if our client is calling and calling a method on the null object, it should not cause any side effects. However, doing nothing can mean different things in different implementations. And this is especially true if the method that we are implementing is supposed to return some value. Okay, so returning a value from the null object could mean we can either return another Null object or we can define return a predefined default value. For example, our method is supposed to return an INT. We can return 0 or 1. If we are supposed to return a boolean, we can return either a true or false. But what exactly is return either true or false? It depends on the implementation. Okay. So there is no common rule that says that if it is boolean always return false. This is not the case. You will have to go study your application logic and find out what a default value will make sense for a null object. Now the code which creates objects of our from our implementation will create and pass our null object in a specific situation. Okay. So if we are depending on some criteria to decide which subclass to instantiate and if we get a value for which there is no subclass, then we can create an object of our null class and pass that null object to the caller. All right, so this is how we can implement a null object design pattern.
