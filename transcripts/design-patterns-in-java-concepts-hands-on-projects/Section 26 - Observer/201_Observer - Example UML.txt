# Observer - Example UML

Now let's look at the UML diagram of the example that we're going to implement in the next video. So here we have an object of order. So this is our subject. Okay? So whenever something changes in our order, we are going to notify all the listeners which are registered with us. So since we have only the concrete object here, we have the attached detach methods which accept an object of order observer. Then we also have methods that can be used to query state values of our order object. Then we have order observer interface. So this is our observer interface and it has a method called as updated where an order object reference will be passed. We have two concrete implementations for our observer interface. First of all, we have a Price Observer, which is going to be interested in finding out if the price has changed. So whenever the order calls an updated method on Price Observer, it's going to call back to the order object and find out what is its price and take some action based on the value of price. The quantity observer is another implementation of our order observer and it is going to be interested in finding out changes in the quantity of items in our order. Okay, so it's quite simple design pattern to implement. Now we're going to see how we can implement it in Java.
