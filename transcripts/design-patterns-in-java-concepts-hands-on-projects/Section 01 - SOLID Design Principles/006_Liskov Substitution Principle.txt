# Liskov Substitution Principle

All right. Now let's talk about the next principle, which is liskov substitution principle. This is the third principle from the Solid Design Principles Group. Now, this principle states that we should be able to use subclass or child class object wherever base class object is expected. And this substitution should not alter desired properties of the program. Okay. It sounds fairly simple actually, and it is quite simple. But remember, we are not only talking about simple type based substitution where we are following the language rules of the, for example, Java, but we are also talking about behavioral subtyping. That means if the base class object was providing a specific behavior and if that base class object is now substituted with a child class object, that behavior should not alter. Okay, so what is expected to happen in that particular code should happen even if we change the base class object with the child class object. Now, this principle is going to need a code to understand fully what the implication here is. Okay, so we are going to look into an actual Java code to understand what Liskov substitution principle is telling us about.
