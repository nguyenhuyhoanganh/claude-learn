# State - Comparison with Command

All right, Now let's compare our state design pattern with command design pattern. So the first difference is that state itself implements the actual behavior, which is specific to a state value for a particular object. The command, on the other hand, doesn't implement any behavior itself. It simply calls a specific operation on the receiver object. So State will actually have the logic or the code which defines the behavior of an object. Command implementations do not write any code or any behavior, but they simply call the receiver and call a specific method on that object. Now a state object represents a particular state value of our context object. So that is what a state object represents. A command object, on the other hand, represents an operation, an action or a request without any direct relationship with the state of receiver. Okay, so the objects themselves represent two different things. All right. So this is the difference between state and command design patterns.
