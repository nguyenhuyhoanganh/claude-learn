# State - Example

All right. Now let's look at an example of state design pattern that can be found in real world examples. So one of the example of this design pattern being used is in the Java server faces. So in JSF, if you're familiar, we have a life cycle interface and implementations of JSF specification will provide an implementation of life cycle interface. Now the main servlet in JSF, which is the Faces servlet, it will invoke the execute and render methods on the life cycle. Now life cycle in turn uses multiple phases in order to execute a JSF request or to process a JSF request. Here, each phase that is used by the life cycle in its execute method represents a state in the state pattern. So JSF has defined six phases which are used by this life cycle interface. So we have phases like restore view phase, apply request values. So depending on whether you are whether the user is requesting a JSF page for the first time or whether he is doing a postback, different states will be executed by the life cycle. All right. Now the life cycle also provides an object of phases, context to these phases so that they can use values or stored values in that particular object. So in this way, the life cycle and different phases in the JSF request processing are examples of state matter. Now if you are not familiar with JSF technology, then you can always give the example that we studied where we saw how a request cancellation can be processed using the state design pattern.
