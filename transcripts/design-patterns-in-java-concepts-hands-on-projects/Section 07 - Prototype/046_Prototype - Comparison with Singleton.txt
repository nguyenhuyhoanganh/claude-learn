# Prototype - Comparison with Singleton

Now we are going to compare the prototype design pattern with another Creational design pattern known as Singleton. Now, we haven't yet seen Singleton, but the chances are that you already know what a singleton design pattern is. A singleton design pattern allows you to have only one instance of a class in your entire code. All right. So what are the differences of prototype with Singleton? In prototype, we return a copy of an instance that means we get two different instances of our class in Singleton. However, we always return the same instance of our class. That means there will be only one instance of our Singleton class. Now, some or even all of the state of the instances that is created using the prototype pattern can be different. So we have seen that when we clone, we make a copy of our state and that state can be mutable. That means we different instances of prototype can have different state. In case of Singleton, since it is only single object that is shared everywhere, the state of that singleton is going to be same no matter where we access it. All right. So these are some of the differences of prototype and singleton.
