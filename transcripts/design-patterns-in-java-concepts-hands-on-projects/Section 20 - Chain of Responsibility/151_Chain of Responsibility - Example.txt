# Chain of Responsibility - Example

All right. Now let's look at a real world example of chain of responsibility design pattern. And as I have already mentioned, the best example of this design pattern is servlet filters. Here, each filter gets a chance to handle incoming requests, and once its work is done on that request, it passes down that object to the next filter in the chain. So all servlet filters are classes that implement the filter interface, which is defined in Java servlet package. Now this interface defines a method called as do filter, and here is the signature of that particular method. So do filter method will give access to the incoming request, the response as well as a special object called as filter chain. Now implementations of filter will use that filter chain object to pass the request to the next object in the chain of responsibility. Now to do that, there is a method called as do filter, which is defined in the filtration class and you have to pass the request and response to the next object or next handler in our chain. Now, in servlet filters, it is common practice to allow other filters to process or handle the request. Even if our current class or current filter has taken some action on that request. Because in the Gang of Four description, you will see that we will pass on the object or request to the next handler. If we are not processing the request in servlet filter all classes are all handlers in chain of responsibility. Get a chance to process or work on that incoming request.
