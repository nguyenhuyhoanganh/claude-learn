# Chain of Responsibility - Implementation Steps

Now let's see some of the steps that you would typically follow while implementing Chain of Responsibility design pattern. So we start by defining our handler, which can be an abstract class, a concrete class or an interface. Now, the main requirement here is that the handler defines a method that accepts the incoming request. So the request can be an object of a class, or it could be a primitive data type. Anything that you need or anything that represents a request for operation. Now, Handler can also define a method that accesses or that provides access to the successor in the chain. Okay, It is not mandatory, but you can add this method which can give access to the next object in the chain. If it is an abstract class, then we can even implement this functionality of calling the successor in our chain in the abstract class itself. Next up, we need to implement the handler in one or more concrete handlers. Okay, so next comes the implementation part. Now, concrete handlers should check if they can handle the request. So based upon the incoming request object, they can decide if it is something that they can handle. If not, then they should pass on the request to the next object in the chain. Now we have to create our chain of objects next. Okay, so we have more than one implementation. So we will be creating objects of these implementation. Now we need to connect these objects together. We can do that in the client code itself, but typically what you will find is that maybe there is a framework like maybe the servlet filter, or it can be an initialization code that is written by you. So there will be a separate code that creates these objects, change them together, and then it hands the first object to the client code. Okay, but this step needs to be done. The creation of chain needs to happen. The client needs to know about the first object only and it will pass the request to that first object. All right. So these are some of the steps that you will follow while implementing Chain of Responsibility Design Pattern.
