And the reason we are using Java effects is that we are better able to see how the objects are collaborating and how the object states are immediately reflected using the mediator design pattern. So in case you don't know, Java is a graphical user interface framework that is provided with Java and with recent versions of Java. The Java has been redesigned and it is now much more usable and it is much more powerful. So coming back to the actual design pattern that we are studying here, the mediator. So we are going to create a single concrete mediator called as UI mediator. So we are not going to go into creating abstract mediator and then implementing it in a concrete class. We are directly creating UI mediator to keep things simple. Now, this example in this example, we are going to create an interface called as UI control and make all the objects that are participating in the collaboration implement this interface. Now this is not what mediator design pattern requires. It is something that we are doing in order to simplify our implementation of UI mediator because the UI mediator is perfectly capable of dealing with classes which do not have any common interface or common base class. Having a common interface, however, helps you to simplify the implementation of UI mediators. So that's the only reason we have this UI control interface which will represent all the colleague objects. Our implementation will be much more simplified. So what is this example? So we are going to have three UI controls in our application. We're going to have a slider which user can change or which user can modify to change a value. We are going to have our regular old text box where you can type some exact value and then we are going to have a label which displays some text. So if slider or textbox changes because of the user interaction, so let's say user types, some value in text box, then the slider control and the label are going to update themselves to show the value that is entered in text box. Same thing is going to happen if user modifies slider. So if we slides our slider control to left, then our value is going to increase and text box and label are going to update themselves to show the new value. And this is an example of how object collaboration happens without mediator. Slider would have reference to the textbox and label because it needs to tell these two controls that it's value has changed and TextBox would know about slider and label, so on and so forth. So using mediator, all these three classes or objects of these three classes only know about our mediator and they will tell mediator when something changes and a mediator will take care of notifying remaining three objects of the state change. So in the UI mediator, we have a simple method called as register using which these controls objects of these classes are going to register themselves with the mediator. We have another method in mediator called as Valuechanged. Now this method will be called by these controls if their state is changed because of user interaction. So if the user types something in a text box, text box is going to call valuechanged on the UI mediator. So how are we going to deal with the actual value? How are the remaining controls going to find out the value they're going to find out using the common interface. So in the UI control interface, we have a method called as control changed and get value. So whenever slider, for example, notifies UI Mediator that its value has changed, it's going to pass itself in this method. So we have a parameter here which is of UI control type and mediator is going to pass on that UI control reference to the remaining objects. And these objects can call the get value method in the UI control interface to get the actual updated value. Okay. So that's why we have the get value method. The control changed method in UI control interface is called by UI mediator to tell these controls that somebody has changed. All right, so value change is called by controls to tell mediator that their state has changed control changed is called by UI mediator to tell these controls that somebody else has changed.
