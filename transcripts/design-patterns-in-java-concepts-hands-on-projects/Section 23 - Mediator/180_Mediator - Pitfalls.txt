# Mediator - Pitfalls

All right, Now let's discuss a couple of pitfalls of mediator design pattern. Now, one problem that you might have noticed as soon as we started this design pattern is that our mediator is now a central control object. Everything flows through that object, and that means this object is going to get complex. As our interaction interactions become complex or the number of participants are increased. Okay. So that is that is one pitfall of mediator design pattern. Making a reusable mediator is quite a difficult task because we don't want our objects or classes whose objects are interacting to implement any specific interface. Mediator is typically developed so that it is tightly coupled with the objects that it is interacting with. Okay, if we want a reusable mechanism for notifying about state change, then we can use another design pattern called as observer. But as we have just seen, observer is for one to many communication arrangement or pattern.
