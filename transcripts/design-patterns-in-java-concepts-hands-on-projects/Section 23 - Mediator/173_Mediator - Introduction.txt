# Mediator - Introduction

Now we are going to learn about mediator design pattern. Okay, so this design pattern helps us remove the complexity between object interaction or object collaboration. So typically in your application, you will frequently find yourself in a situation where an object, whenever it changes its states, it notifies few other objects about that state change. And these objects can take some action based on the new value of state or based on which field of the state has changed. And these objects in turn, will notify that when their state is changed. Okay, So we have this communication between these group of objects and whenever state of any one of these objects changes, it notifies the remaining one. Okay, So we have this complex web or complex interaction pattern between these objects and mediator. Design pattern encapsulates that interaction in a single object. Now, due to this encapsulation, what we achieve is loose coupling between these objects, because you will find that typically whenever an object wants to communicate with another object, it has to know about that object. So it will need a reference to that object and it will call a particular method on that object. So that means we have this tight coupling for calling a method and we cannot do anything about that with regular objects. However, with mediator, what we can do is we will have a single object and called as mediator, and all the objects will only know about our mediator. So in our group of objects, if its state changes, it will tell the mediator that its state has changed and mediator will take care of notifying all the other remaining objects that you know. This object has changed its state. In the similar way, if somebody sends a notification to mediator, mediator will tell our object that, okay, somebody has its state change. All right. So mediator acts as this kind of a black box. And our object only knows about this single object mediator, and it doesn't care how many objects are participating in that interaction or collaboration. So you will see a benefit of this arrangement in that if, let's say we want to remove an object from our group of objects. So we will need to only change the mediator because mediator has now the references to these objects. Okay, so only mediator has references.
