# Memento - Pitfalls

All right. Now let's look at the pitfalls of memento design pattern. And the first pitfall is creating the snapshots. So creating a snapshot of an object state may not be simple if there are other objects which are part of our originator, because now we have to actually clone those objects so that we have a true memento or a true isolated state in our Memento object. And on the other side, resetting our originator is also not that easy in every scenario, because if our originator is tied with other classes and whenever our originator changes state, those objects also get affected. Then simply copying the older values in our originator is not enough because then those objects are going to be in an invalid state. So reset operation also is not easy in every scenario.
