# Memento - Comparison with Command

All right, Now let's compare our Memento design pattern with command design pattern. Now, there is actually no similar design pattern that does what Memento does because Memento stores an object state so that we can reset it. There is no design pattern that we have studied so far which does exactly similar thing. However, since Memento is used with command design pattern, it's important that you understand differences between these two. The obvious difference between the intent behind these design patterns. Memento itself stores only the object state while command encapsulates an actual operation or an action. Apart from that, the state of Memento is completely sealed for everyone except the original. No one can even read what is stored in a memento. That's the requirement of this design. Commands. However, they are typically immutable. That means we give everything that is needed by the command in the constructor. But the state of the command itself is often readable, so we provide some read only access to these commands and memento. The requirement It's mandatory that we do not have any getter methods in command. There is no restriction on the having a getter method or allowing the read only access to its state. Next. A memento needs to be stored for it to be of any use, because if we create a memento and simply let go of it immediately, then there is no point in creating a memento. If we store that memento, however, and use it in future to reset our state, then it has some value. So storing a memento object is mandatory for it to be of any use command. However, they are also typically stored. However, you should understand that this is an optional feature, so once our command is executed, its job is done. If you are not using any undo functionality, then there is no requirement to store these commands and they can still be useful. So storing of the objects of memento is required. Storing of command objects is completely optional and we can use it as an additional feature of command.
