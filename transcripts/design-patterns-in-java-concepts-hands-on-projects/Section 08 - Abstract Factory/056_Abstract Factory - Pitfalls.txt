# Abstract Factory - Pitfalls

Now let's look at the pitfalls of abstract factory design pattern. One thing you will notice that it is a lot more complex to implement an abstract factory than the factory method or any other creational design pattern. Whenever we have a requirement to add a new product to our family of products. As for example, in our example, we had a compute resource and a cloud storage. Let's say we wanted to add a network, private network or a virtual network. Then we need to modify all of our factory implementations as well as the base class. All right. So this is a drawback of using abstract Factory. Now, typically it is quite difficult to visualize need for Abstract factory when you are just starting your applications design. All right. So this this pattern typically starts out as a simple factory. And then in the development, you realize that, okay, we can use the abstract factory design pattern. Now Abstract factory design pattern, as you can see, is very specific to a problem. And that problem is having the product families or having objects that work together. All right. So it has a very specific application and typically you will not find it to be generalized to solve other problems. All right. So these are the pitfalls of abstract factory design pattern.
