# Abstract Factory - Comparison with Factory Method

Now let's compare Abstract Factory with factory method. So in factory method, what we want to do or what we achieve, in fact, is that we isolate our client code from the objects that it is using. In Abstract Factory, however, we hide the factories which create these objects as well as the objects themselves. They both are hidden or isolated from the client. Our client works on interfaces for factory as well as the products that it is using. Factory method is typically concerned with one single interface and its implementation or one class and its subclasses, so it can create any one of those from that single inheritance hierarchy. Factory method is not concerned about how that product works or collaborates with other objects in our application in Abstract Factory, however, whenever we have multiple objects that form a set or kit and they are supposed to be working together, then we can use this abstract factory design pattern. All right, So these are these are the couple of differences between abstract factory and factory method.
