# Object Pool - Example UML

All right, Now let's look at the UML diagram of the example that we're going to implement in Java code. So we start off on the far right side. You will see we have a interface called as pool. So this is this interface is going to be what our object pool is going to use. So we can have multiple classes that can implement this pool level interface and they would be eligible to be pooled in our object pool implementation. Now this interface simply defines a single method called as reset. And the purpose of this method, of course, is to reset the internal state of the pooled object. Now we are going to have an interface called as an image, and this is going to represent an image that we we have loaded from the disk and we want to render it on multiple times. And then we have a concrete class called as Bitmap, which is an implementation of image, and it has its own state. And now the state is going to be the image data, which is going to be what is loaded from the disk and location and extent. So these are the properties are part of its state that need to be reset before it can be reused. So we can reuse the image data because that's why we are going to cache this bitmap because every time an image instance is created, we don't want to read from the disk. So that's why we are going to pool this particular bitmap. So we're going to keep the image data as it is and we're going to reset its location and extent. Then we're going to have a object pool class. Now, again, this is going to expose two important methods. The first one is the get method that is going to return the pooled image object, and then it is going to provide another method called as release, which is going to take an instance of this particular bitmap and it's going to add that back into the pool. All right. So this is the implementation that we're going to implement in Java.
