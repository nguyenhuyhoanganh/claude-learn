# Object Pool - Implementation Steps

All right. Now let's look at some of the steps that we need to follow in order to implement object pool design pattern. So we start by creating our class for object pool, but it must satisfy a couple of requirements. First first requirement is that it should provide a thread safe caching of objects in memory. Next, it should provide methods that client can call in order to get object from the pool. And another method which client can call in order to return an unused object. Now, one thing that this class must take care is that it should reset the object before returning it to the client. Now that means our reusable object or the class whose objects we are going to pool, it must provide that reset method. Now there is one choice that you have to make while implementing your object pool. About what happens when your pool runs out of objects. So you have two choices. You can either create new object if your pool is empty or you can keep on waiting until some object becomes available. Now, this choice is heavily dependent on your implementation. So if your class object is tied with an external resource which is limited in nature, then of course you have to wait. Or if you have access to unlimited resources, or if your object doesn't depend on any external resource, then you can keep on creating new objects and then caching them when they are returned to your pool.
