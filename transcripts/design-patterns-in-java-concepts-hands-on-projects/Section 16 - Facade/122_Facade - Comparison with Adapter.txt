# Facade - Comparison with Adapter

All right, Now let's compare our facade design pattern with adapter. Now you may find them to be somewhat similar because adapter also provides an interface which client code uses and facade also provides an interface which the client code uses. But there are some great differences between these two design patterns. So first of all, the intent behind facade is to simplify the usage of our entire subsystem. Facade is not written for one single class. Okay. So for example, if you have one class, you won't write a facade, just use that single class. It is typically used when you have a subsystem which consists of multiple classes and interfaces, and you want to simplify how the client uses our subsystem. You would use facade design. Pattern adapter, on the other hand, is meant to simply adapt an existing object to a different interface. Okay, so Adapt works on a single object, and the purpose or intent of adapter design pattern is to adapt that object to a different interface. Next difference is that facade is not restricted by any existing interface. So when you are designing facade, you can write any method that you want. You will have only one purpose or only one goal. That whatever operation we define in facade is easy to use. You don't have any restriction that you have to implement this interface or you have to follow these particular method templates. There is no such restriction. Adapter, on the other hand, is always, always written to conform to a particular interface which the client code needs. Okay, so this is another difference between facade and adapter.
