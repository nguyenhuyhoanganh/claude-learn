# Facade - Implementation Steps

All right, Now let's see what we need in order to implement the facade design pattern. So we start by creating a class that will serve as a facade, the class which will be used by our client. And we define operations, our methods in the in this particular facade class. But we take into consideration the overall use case or task that our subsystem is designed and implemented for. For example, if you have a subsystem that is responsible for, let's say, printing documents, then you will have a simple method that takes an object of a class or a file, a file input stream object, and it simply prints that content of that particular file or input stream. All right. So we are thinking about use cases, overall use case that the client code is interested in. We are not going to write methods that correspond to another method which is part of a class in our subsystem. All right. And that is the important part here. That facade should be readily usable and it should expose our use cases, our tasks as simple operations. All right. And this method, which we have defined, it takes care of interacting or calling into our subsystem and using these multiple classes and interfaces, which are part of our subsystem. Now next, we are going to see how we can implement this by looking at the UML of example.
