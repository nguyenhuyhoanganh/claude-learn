# Facade - Example UML

All right, now we're going to look at the UML of the example that we are going to implement. So in this example, we are going to have an email subsystem. Of course, this subsystem will be not sending an actual emails, but it represents all the classes that you would typically use if you are sending an email. So in this email subsystem we have classes like Mailer, email, email, builder, stationery and all sorts of classes. So if our client wanted to send an email using our subsystem and there was no facade, he client code would interact with all of these classes one by one. And that means this particular code now becomes tightly coupled with all of these inner classes of our subsystem. To solve this problem, we're going to create an email facade which has a simple method that corresponds to the use case which is implemented by our email subsystem. And that use case is send order, email. And this method is going to simply take an object of order class, which we are going to implement, and it's simply going to tell whether the email was sent successfully or not. So Client now only interacts with the email facade. He doesn't need to know what exactly are we using inside this subsystem. All right, so let's jump into Java code and see how we can implement it.
