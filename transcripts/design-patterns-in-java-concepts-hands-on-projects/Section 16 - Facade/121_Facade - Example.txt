# Facade - Example

All right. Now let's look at an example of facade design pattern in Java class library and the Earl class, which is present in the Java dot net package is a great example of facade. We use this class Earl class in order to represent a unified resource which is present on the network, which can be a URL that we can type in the browser, or it can also point to a resource that is present on a file system. Now this class provides a method called as Open Stream, which returned, returns us an input stream to the resource which is pointed to by the URL object. Now opening a stream to a remote server requires interacting with a lot of different classes and all that complexity is hidden by the URL class. And that is why this is a great example of facade design pattern. All right, so this class takes care of dealing with multiple classes in the Java Dotnet package as well as dealing with some internal son packages. And this is what facade design pattern is, because there are two main goals or two main purposes in the when using facade design pattern. The first goal of facade is to reduce the coupling between the number of classes in a particular package and the client code. Okay. We do not want our client code to be coupled with multiple classes from a package. That is the first goal. Second goal of facade is to simplify usage of the functionality that is provided by those classes in a particular package. And Earl class covers both of these goals beautifully. Now let's look at the UML diagram of these classes. So we first of all, we have the client code that is going to interact with our Earl class. Now this class is actually what is a facade. This is part of Java dot net package. Now when we communicate with Earl class and call the open stream method, this class is actually dealing with multiple classes which are present in the Java dot net package, which are again dealing with some other classes which are internal to Java class library. So if we are using Earl, we are simply going to create the object of this class and provide the URL to the remote resource. And then we are simply going to call the open stream method, which will return us an object of input stream, and then we can use that input stream to read data from that server. So you will see that the from the perspective of the client code using this class is very easy. We do not have to deal with multiple classes. We only deal with the single URL, which is the facade. Now if you take a look at the code which is present in this particular class, you will see that it deals with multiple classes internally. So this second code snippet is taken from the Earl Dot class file from the jar. And here is an example of a method that is present in this class. So we have a method called as get URL stream handler and you will see that this class is internally dealing with classes like URL stream handler. It is dealing with particular factory in order to get the particular type of protocol handler and it is dealing with security. So all those classes are not coupling with our client code when we are using the URL object. All right. So this is a great example of facade design pattern.
