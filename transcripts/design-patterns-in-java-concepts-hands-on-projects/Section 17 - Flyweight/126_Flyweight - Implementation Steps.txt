# Flyweight - Implementation Steps

So let's look at the typical steps that you will follow while implementing a flyweight. So our first job is going to identify the intrinsic and extrinsic state of the object for which we are writing our flyweight design pattern. We will create an interface for our flyweight that defines all the common methods. However, that those methods will need an object or a value for extrinsic state that can be used by our object. Now in implementation of shared flyweight. So that is the flyweight object or flyweight class whose objects will be shared. We only add the intrinsic state inside that class definition and we also implement the methods in that class. And we expect that the extrinsic state will give us the remaining information to carry out or provide the functionality that we want in unshared flyweight implementation. We simply ignore that we have any extrinsic state because all the state that is needed by our class object is going to be provided in that class. However, you are still going to have these methods that pass some extrinsic value. So you should typically ignore these values because you have all the information that you need in your object itself. Next, we implement the flyweight factory and this flyweight factory will cache the shared flyweight and it will also provide a method to get these objects. So client will be working with the flyweight factory so it doesn't need to know which objects it should share and which objects should be created newly. The flyweight factory handles it for the client. Now in our client we either maintain that extrinsic state so we can have these values stored in our client code, or we can compute them on the runtime and use them whenever we want to call an operation on our flyweight.
