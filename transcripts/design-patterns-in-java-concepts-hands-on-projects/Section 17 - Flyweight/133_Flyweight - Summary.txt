# Flyweight - Summary

So here is a summary of flyweight design pattern. We use this pattern whenever we have large number of objects of a particular class and we have a performance concern that we cannot maintain these many instances in our memory. Now we can use this pattern if we can separate out our object state into two parts. So we need to have a state which is called as intrinsic state, which remains or which can be shared in any context where our object is used and a part of state which can be externalized, which is called as extrinsic state. Now, flyweight class itself only defines the state which is intrinsic. And the this is the state that can be shared in any context where our object is used. Now, code, which is using the flyweight instance, it provides this extrinsic state when calling any operation on the flyweight. So flyweight object combines these two state. It has the intrinsic state within it and it gets the extrinsic state in each operation combining these two states. Our flyweight object can provide the functionality which is needed by the client code. Now, the client code itself can decide whether it can whether it wants to store the extrinsic state value or it can calculate the extrinsic state every time. It needs to use any operation which is defined in our flyweight. Now let's look at the UML diagram of this design pattern. Now, each operation has a parameter for providing the extrinsic state, and this state is given to us by the client code. Then we have concrete flyweight. So this is a class whose objects are shared in our good. This class only defines the intrinsic state within itself and it provides implementation for these operations. These operations accept the extrinsic state value, which is then combined with intrinsic state to provide the functionality of this class. We can also have unshared concrete flyweights. So these are our regular classes which do not support sharing of their state. So in these implementations we have all the state information defined in the class itself. And the operation which provide the extrinsic extrinsic state are ignored. Those arguments or parameters are ignored by this class.
