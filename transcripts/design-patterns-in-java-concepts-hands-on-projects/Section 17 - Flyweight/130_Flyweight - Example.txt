# Flyweight - Example

All right. Now let's look at an example of flyweight design pattern in Java class library. And the first example that we're going to look at is the wrapper classes. So Java has provided us with wrapper classes like Integer, short byte, Character, Boolean, float and double. These classes represent our objects of these classes are representing the primitive values so that we can use these classes wherever an object is expected, especially when working with collections. So these wrapper classes, some of these wrapper classes make use of flyweight design pattern. The classes which use this flyweight design pattern are integer, short byte and character because you will see that these classes have capability inside of them where they can cache some objects and return those objects whenever we want. So for integer short and byte wrapper classes, if we ask for a wrapper object which is within the range of -128 to 127, we will get an already created object. That is, we will get a flyweight because that is a shared object for character wrapper class. The range is from 0 to 127 I believe float and double wrapper classes do not cache any value. Okay, so double and float wrapper classes do not use flyweight design pattern. However integer short byte character will use flyweight design pattern and this pattern will be used only if we use the value of static method which is defined in these wrapper classes. Because this method value of method works as a factory method for our flyweight. So this method will check whether an object is whether the required object is already present in the cache or not, and if it is present then that object is returned. So here is a code snippet that is taken from the integer class file. So in the value of implementation you will see that the passed argument is checked. Whether that falls within the range that I just described, which is -128 to 127. If the primitive value is within that range, then an already created integer object is returned from this integer cache. However, if the argument is outside of that range, then a new integer is created and we are not using any flyweight. All right, so this is an application of flyweight design pattern in these wrapper classes. And once again, float and double. Do not use this particular design pattern. Another example of flyweight design pattern that you will find in Java is string pool. Now, string pool is a special area in the memory that is allocated at runtime and it is used by JVM to cache string literals. So if you have defined multiple strings in your code using double quotes, then by default those strings or string literals are kept in that string pool so that they can be reused. We can also call internal method that is present on the string class to explicitly request JVM to intern or put our object in the pool. However, whenever we call internal method, Java is going to first check our pool and see if the object is already present in the pool or not. If there is an already present string which matches with our string object, then we are going to get a reference to that cached object. That is, we are going to get a reference to that flyweight. However, if the object is not present, then Java is going to put that object in the cache and make it a flyweight and return that reference to us. So string pool, which is used by Java is an example of flyweight design pattern.
