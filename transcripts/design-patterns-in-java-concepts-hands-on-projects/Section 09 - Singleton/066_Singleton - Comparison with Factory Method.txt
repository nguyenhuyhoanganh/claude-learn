# Singleton - Comparison with Factory Method

Now let's compare our Singleton design pattern with factory method. Now the intent behind Singleton design pattern is to ensure that only one instance of our class is ever created. The factory method, on the other hand, has main concerns related to separation of client code from the object, creation, logic and delegation of object creation to subclasses. So the purpose behind these two design patterns is quite different. Even though both of these belong to Creational design Pattern category, a singleton instance is created without need of any arguments from client code. So as we saw, we have a single method called as get instance that is called anytime we want access to our singleton instance and that method doesn't need any arguments. We can parameterize the object creation in a factory method. All right, so these are the differences between singleton and factory method pattern.
