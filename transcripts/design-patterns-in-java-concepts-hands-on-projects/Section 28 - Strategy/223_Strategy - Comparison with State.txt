# Strategy - Comparison with State

All right, Now let's compare our strategy design pattern with state design pattern. So in strategy design pattern, we create a class per algorithm. So if we have two different algorithms that solve the same problem, we create two separate classes. In case of state design pattern, however, we have a class per possible state value that our object can take. Then next difference is that strategy objects typically do not need to know about each other because they are completely separate algorithms and they don't need to know about what other algorithms are implemented in case of state design pattern. However, if our states are responsible for triggering state transitions, then they need to know what other states are available and they will create an object of that new state and then they will save that state in our context object. All right. So these are some differences between strategy and state design pattern.
