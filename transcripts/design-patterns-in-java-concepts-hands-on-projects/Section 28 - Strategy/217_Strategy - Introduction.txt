All right. Now let's talk about strategy design pattern. This is a very simple design pattern and it would be a nice break after learning about state design pattern. So this design pattern allows us to encapsulate an algorithm in a class, and that's its job. So what we can do is we can configure our context or a main class with an object of that class, and it will start using that algorithm to carry out any given operation. And this is all that this design pattern allows us to do. So and this is really helpful if we have multiple possible algorithms which can be used to solve a given problem. So you might have learned about sorting algorithms. So if you have a list of integers with you and if you want to sort them, you can use either a bubble sort and insertion sort, merge sort. There are various possibilities. Now we can implement each algorithm in a separate class and give an object of that class to our main class, which can then use that implementation of algorithm to sort through our list. But remember, these algorithms are not limited to sorting, sorry, sorting, and searching. We can solve our regular problems. We use algorithms to solve our regular problems which are not necessarily related to sorting and searching and a good indication of applicability of this strategy design pattern.
