# Strategy - Implementation Steps

All right. Now let's look at a sequence of typical steps that you would follow while implementing this design pattern. So we start by creating our strategy interface. So remember, implementation of that interface are going to be algorithms which solve a particular problem. Now, this interface is used by our main or context class, and that class is going to provide our strategy with any information or any data that it needs. So typically, your strategy should not have any state in it or it should not have a need to store any input inside of it, because context is going to provide our strategy with all the data that it needs. Then we need to provide implementations for this strategy interface. And each implementation is basically a separate or a different algorithm. Now we will implement a class per algorithm, so each class of our strategy interface, each class implementing strategy will represent a separate algorithm. Now our context class provides a way to configure it with a particular strategy object. So there should be a way for the client to tell our context that use this particular strategy object. So the object itself, the strategy implementation, which implementation to use is decided by client and not by our context class. That's the important point to note here. All right. So this is how we can implement strategy design pattern.
