# Composite - Example UML

All right, Now let's look at the UML diagram of the example that we are going to follow in order to implement the composite design pattern. So first of all, we have an interface here called this file, an abstract class called as file, and this will be our component. So we are going to define methods that are common to both our leaf node and composite node in file. What I've done here is I have defined the operations that allow you to add children in this particular base component class. And of course, I'm telling you that this is not the only way you can have these operations defined in your composite. So next up, we have binary file which represents our leaf node, and it has provided the implementation for methods that are applicable. It's going to also provide implementation for methods that do not apply to it. Directory is a composite here, so this class will provide the implementation for adding and removing children. Any operation that we actually call on this directory will should be delegated to the children. Next up, we have the client, which is going to use our file interface in order to work with our tree structure. All right, so this is how we are going to implement a composite in Java code.
