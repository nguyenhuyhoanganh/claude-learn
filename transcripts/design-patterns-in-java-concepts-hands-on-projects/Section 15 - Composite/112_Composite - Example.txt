# Composite - Example

All right. Now let's look at an example of composite design pattern. Now, after studying this design pattern, you will get an idea that there are many applications of this design pattern when working with UI frameworks. Because in UI frameworks we often run into situations where we have a tree of UI controls or where we have some sort of container where other UI components like buttons, text boxes are added. And in those situations, composite design pattern is really useful. And that's why the example that we are going to look at for this design pattern is from the Java Server Faces framework. So in JSF we have a class called as UI View Road. Now if you have worked with JSF, you will find that this class is used and this class can contain other JSF components, UI components inside of it. So we have components like UI output, UI message which act kind of like a leaf node in the composite design pattern. So let's look at the UML diagram of only these classes. So in JSF we have a common interface called as UI component, which is implemented by all UI components in the JSF framework. Then we have a common base class called as UI component base. And in this base class we also have methods that are related to child management. So we have a method called as Gate children Gate Child Count, which are actually implemented or useful when working with UI view. So UI view route in the JSF is actually an example of composite design pattern and this class are object of this class can contain other objects of UI component type UI output, UI message and many other classes are acting like leaf nodes in the composite and in the implementation of gate children. It will not return anything in the gate. Child count these objects will return zero. All right, so UI view route in JSF is an example of composite design pattern.
