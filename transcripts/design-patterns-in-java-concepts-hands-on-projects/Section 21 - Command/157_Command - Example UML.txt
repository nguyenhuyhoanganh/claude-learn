So first of all, we are going to have our command interface. It has a simple execute method defined in it. We're going to have a few implementations for our command interface. And the whole premise of the example is that we are going to work on exchange web service. So exchange Microsoft Exchange is a email server, mail server, and you might have heard about distribution list. So these are these are collection of email addresses that are created on mail server. And we can send email to that distribution list and that email will be delivered to all members of that list. Now, of course, we don't have a license to use an actual exchange server and call the real exchange service. So what we are going to have is at the bottom you will see we have service. So this is a dummy implementer, Asian, and we are going to assume that this service actually talks with the real exchange web service. Okay. So this is the receiver object and our commands are going to call methods on this receiver. So, for example, we have a concrete class here called as Add member Command. So this command is going to take a reference of service. Okay. So it will need a reference to our receiver and it will need an argument, for example, email address. And in this execute implementation, it's going to call the Add member method on our exchange web service and the similar lines. We are going to have a remove member, so on and so forth. So you get the idea how our example is going to work. Now client code is going to be in charge of creating instances of these concrete commands. Once an instance is created by the client, it's going to be sent to another class called as Background Runner. Now we are going to implement a simple thread here. Nothing fancy. We are not going to handle all the scenarios or corner cases, but this class is going to demonstrate how a separate thread can queue these command instances and call them at a later point of time. Okay, so Client is going to create an instance and send it to a queue which is present in our background runner and our thread is going to simulate some wait time between these commands and call these commands one after another.
