# Command - Implementation Steps

Now let's look at a set of steps that you would typically follow while implementing the command design pattern. So we start by writing our command interface. And once again, this is going to be a very simple interface. It should only expose the execute method, and this method should not need any arguments to be called because our command is going to be executed in a different part of our application. So whoever is creating our command object is not going to be the same code which is executing that command. Okay. Next up, we need to implement this interface in a couple of concrete classes. Okay. And each class that we implement knows exactly which method it is going to call. Okay, so let's say we have an object called as, let's say, image and we want to create commands for these manipulating this object. Then we will have a simple command interface and we will have concrete classes, for example, like resize image and this concrete implementation knows that it needs to call resize method on the image. What it needs additionally is that it needs to know what is the new size. Okay. So we need to provide it the concrete implementation instance. We need to provide it with arguments that it will use to call that operation. Okay. Now some command implementations also support the undo capability. So you might have seen that in desktop applications where you can undo the previous set of commands that you have carried out. Okay. So if you want to provide that type of functionality, maybe you are designing a workflow designer which is run in your in customers browser, then you can provide them with undo functionality. Okay. And it is quite easy to do that with command design pattern. Now of course it needs a help from the actual object on which the command object is calling methods on, but still using the command design pattern. We get there pretty fast. All right. Then each concrete command, as we discussed it, knows exactly which operation it needs to call and we need to provide it with the arguments. Okay, so in the constructor of our concrete command, we should accept the object on which methods will be called, and it should also accept arguments that are used while calling that method. Now client code is in charge of creating these command objects. And it will provide the receiver instance or the object on which methods will be called, and it also provides it with the arguments. Okay. And all this information is sent to our concrete command instance, and that instance is now ready to be sent to different part of our application where it will be executed. Okay, So this command instance is typically sent to different part of our code where the actual invocation of execute method happens. All right. So this is how you will implement a command design pattern. Next up, we are going to see the example that we are going to implement in Java.
