# Template Method - Implementation & Design Considerations

All right. Now let's talk about some implementation and design considerations that you should think about when using template method design pattern. So the first consideration is that a balance must be kept in how much granular our steps are. So if we have too many methods and steps in our algorithm, then overriding or implementing all those methods in a subclass can be a tedious task. And often this ends up creating methods that are doing some very primitive operations. So that is not what we desire here. We want to provide some meaningful steps. However, if we define very few steps, that means the subclasses are going to end up implementing large portions of algorithm at a time, and that means you may lose control of how your algorithm is implemented. Okay, because we want to keep the implementation or the steps in the template method itself. The algorithm is defined there. However, if you keep very few steps, that means you are losing that control and subclasses now have a lot of control on how that algorithm is implemented. Okay, so the behavior might differ between these two subclasses. So you have to keep a balance between these two extremes. Uh, last implementation consideration is that you can mark your template method as final if you want to prevent your subclasses from overriding your actual algorithm method and modifying it so you can declare your template method as final and allow only abstract or allow only steps to be implemented in subclasses. Now let's talk about some design considerations. So the first consideration is that we can use inheritance between subclasses which are implementing our steps in order to reuse some of these steps. Okay. So for example, if you have a subclass that has already provided implementation for these steps, then you can extend from that subclass and only customize a few of the steps that you want. Okay, so that way you won't have to override all the methods or all the steps in each subclass. Now, we have already talked about this, but the factory method design pattern often uses template method. Okay? And the actual factory method is typically called as one of the steps that is taken in that template method. All right. So these are some of the implementation and design considerations while implementing this template method.
