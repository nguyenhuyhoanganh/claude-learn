# Template Method - Implementation Steps

All right. Now let's look at how we can implement this template method design pattern. So let's look at some of the steps that you would typically follow while implementing this design pattern. It's a pretty straightforward process. We start by defining our template method, which contains the algorithm that we want to implement. Now, when we are talking about template method, we are not talking about a generic method or something like that. It is simply a method that defines an algorithm which uses multiple steps. So we are going to break down our algorithm in multiple steps that are executed one after another, and each step becomes an abstract method in our base class. Now, while breaking down our algorithm, we have to ensure that we are not creating very small steps or steps which are very primitive in our algorithm implementation, because then implementing a subclass can become quite a tedious job because we have these lots of methods, but we also want to give enough freedom to our implementing classes so that they can do some meaningful, meaningful work, or it is sufficiently granular that they can carry out their work efficiently. Next, we have to implement these abstract methods in one or more subclasses where we are going to actually provide implementation for these steps. All right. So this is how we can implement a template method design pattern.
