# Template Method - Pitfalls

Now let's talk about some pitfalls of template method design pattern. So the first pitfall is that tracking down the actual code, which is executed as part of our algorithm, requires looking up multiple classes because you now have the client now has choice of which implementation of steps to use. So you may have to look up multiple classes there and this problem becomes really a problem if we have subclasses which are inheriting from each other. So we discussed that we can create a subclass which implements all these steps and we can create another class which extends from this subclass which has already provided implementation of all the steps and override only few of these steps. So if we keep on creating this type of inheritance hierarchies, then if we have a problem or if we want to debug that algorithm implementation, we have to end up we have to look up multiple classes. All right. Uh, unit testing for of the subclasses can also be a problem if these steps assume that some type of state is present in our base class or subclass itself. All right. So while testing these individual steps, you may have to provide actual data or do some processing to get to the real data which is applicable to that particular step. So unit testing can be a little bit more involved while using template method design pattern. All right. So these are some of the pitfalls of template method.
