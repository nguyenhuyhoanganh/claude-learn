# Template Method - Summary

Let's summarize template method design pattern. So this design pattern allows us to define a skeleton of an algorithm in base class. So we only define steps in our algorithm, and these steps are defined as an abstract method in that base class. Now, subclasses of our abstract base class will provide implementation for these steps. So this way we can have different implementations for the exact same algorithm, so we can have two subclasses that carry out a particular step in our algorithm, but they might use some different implementation than each other and client will actually use or create an object of one of these subclasses and then use the algorithm which is defined in our template method. The factory method design pattern is often implemented as part of some template method, so the actual factory method is called inside our template method as one of the steps. A One drawback of this design pattern is that the actual algorithm implementation is now spread across multiple classes. So if you want to understand how this algorithm works, that is defined in template method, you have to look up multiple classes and read code there to understand how this algorithm is implemented. Now let's look at the UML diagram of template method design pattern. It's pretty simple diagram. We have an abstract class here that defines the actual template method. So this method is going to define the skeleton of an algorithm and the steps inside which are needed in that algorithm are defined as these methods which are abstract. So step one, step two, step three are the steps that are called by the template method. And these methods. Step one, two, three are abstract. Then we can have one or more concrete implementations or subclasses which extend our abstract class and they provide implementation for these steps. All right. So client will create an object of the subclass and it will call the template method, which will result in using step implementation that is provided in that particular subclass. Now let's look at a typical implementation of this design pattern. So here we are considering considering an example where we want to print an order object. To do that, we have created an abstract base class called as Order Printer. We have a method here called as print order, which is our template method. And this method, the template method defines the algorithm, which is used while printing the order object. So we have an algorithm here which requires that first of all, a file header needs to be printed, then the order number needs to be printed. Then all the items that are present in that order are printed. Finally, the total and footer for that document. So this is our skeleton algorithm and each step in this algorithm is declared as an abstract method in this abstract base class. All right, so print order is our template method. Then we have a subclass here called as Printer, which extends from our order printer and it provides implementation for these steps. So here, whenever our print order method calls start, we are going to return an HTML header that is applicable to HTML file. In the same way we are going to use HTML tags to format the output that is printed in the file. All right. So this is how template method design pattern is implemented and this is how we implement steps in our algorithm.
