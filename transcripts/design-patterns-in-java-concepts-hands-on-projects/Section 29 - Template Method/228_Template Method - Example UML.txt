All right. Now let's look at how the example that we're going to use while implementing this design pattern. So we are going to take the same example that we discussed when we were introducing this design pattern. And that is we're going to create an order printer class. So we have an order object in our application which had some items in it. It has an order number and it has a total. Our order printer is an abstract class that defines a print order method, which is our template method. So print order method has an algorithm defined in it as a series of steps, and each step is implemented or defined as an abstract method in our order printer. So we have these steps like start the document format, order number, format items, format total, and finally end the printing of our order object. So these are the steps which are used by our print order template method. Then we are going to see we are going to have two subclasses which implement the steps. So we are going to have an HTML printer that's going to format all this information in an HTML document. So it's going to return string, which will which will be HTML formatted. So it's going to use HTML tags in order to format this information. Then we are going to have a simple text printer which is going to simply convert these, convert this information which is stored in our order object to simple strings. All right.
