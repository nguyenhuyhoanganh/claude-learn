# Template Method - Comparison with Strategy

Let's compare our template method design pattern with strategy Design pattern. So the first difference is that in template method, all the subclasses that are that we are going to create, they are all going to implement steps for the same algorithm. So our algorithm is defined once in our template method. And what subclasses are doing are they are they are simply providing implementation for steps in that same algorithm. In strategy, however, each subclass which implements our strategy interface defines a completely separate algorithm to carry out same task. So algorithm implementations are different in each strategy subclass. In tablet method client relies only on inheritance to get a slight variation of that same algorithm. So in our example, we saw that we can create a text printer or we can create an HTML printer and the client was using different subclass, so it was relying on inheritance that is inheriting from our order printer to get a variation of that same algorithm. No matter which printer we used, we are going to get the same content in strategy. However, the typical usage involves configuring another class with a particular strategy object and then the client would call an operation on that object. So we use composition when we are using strategy and we use a strategy object to configure another object which uses that strategy implementation. All right. So these are a couple of differences between template method and strategy design pattern.
