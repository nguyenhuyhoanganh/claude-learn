# Template Method - Introduction

All right, now we're going to learn about template method design pattern. And we have briefly seen this design pattern when we were studying factory method design pattern. All right. So it's time to go into detail and learn what exactly is a template method. So using this design pattern, we can define an algorithm in a method as a series of steps, and each step in our algorithm is basically a method call. And the method itself is is declared as an abstract method in that same class. So what happens now is that subclasses are going to extend from our base class, which has these abstract methods or steps of our algorithm, and they will provide the implementation for those steps. And this is how this design pattern works. You can think of these methods that we are defining the abstract methods as hooks, which are called from our template method. This pattern allows you to defer the implementation of parts of your algorithm because you know that those parts are going to change or they can have different implementation and you are now letting your subclasses decide or choose what kind of implementation they want to provide. So let's take an example of let's say we have an order object in our application. Okay? So we have an object which represents an order that is placed in our on our website. Now, let's say we have a requirement that we want this order object to be printed or exported to a file. Now we know exactly what needs to be present in that file. We know that we need an order number. We know that we need to export the items which are present in the order, and we know that we want order total to be present in that file. What we don't know is the type of file that our client can ask us to generate. So client can ask us to generate a PDF, file a text, file an HTML file. So there are multiple options on how these details can be printed. However, we still know what each of these file is going to contain. Okay, so what? We just listed the order number items in our order and the order total. So these are the contents. They are going to remain the same. So what we can do is we can use this template method design pattern and define our basic algorithm that says that print order number first, then print these items and then print the total. And we're going to call our abstract methods that are defined in our template method so we can write a class which generates a PDF file, and it's going to implement these steps where it is going to actually generate or write into a PDF format. All right. So this is what this design pattern is all about. Now, template method is also an example of inversion of control principle. And this principle you might have heard it described as don't call us, we will call you the Hollywood Principle. So what this means is that we are simply going to provide implementation and somebody else is going to call that implementation. Spring is another example of inversion of control principle. However, spring implements this principle as a dependency injection. So dependency injection is another type of inversion of control. Principle and template method is a separate example of that same principle. All right, so now let's look at the UML diagram of this design pattern. It's pretty simple design pattern to learn. So first of all, we have an abstract class and here we are going to define our template method. Okay? And this template method is going to use one or more steps which are defined as abstract methods in our abstract class. Then we can have more one or more concrete classes which are going to extend from that abstract class and they're going to provide implementation for those steps. All right. So it's a simple pattern to implement and you will find that it is of great use and you will find it used a lot of times in real life applications. All right.
