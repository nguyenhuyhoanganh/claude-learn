# Decorator - Comparison with Composite

Now let's compare decorators with another structural design pattern called as composite. Okay, so composite allows you to have multiple child objects or you can create an aggregate using the composite design pattern. So what are the differences? So instead of decorator is adding to the existing behavior or existing functionality of the object. Okay, so decorator adds to the functionality. The composite design pattern, however, is only meant to provide an object aggregation. Okay, so composite is only meant to support having multiple child objects within a single object. Now decorator can be thought of as a degenerate composite, which has only one component because as we have seen, decorator contains only one copy of the actual concrete object. Composites, however, support any number of components in their aggregation, so they will typically have a collection of objects within themselves. Decorator has only one component object within itself. All right. So these are the differences between decorator and composite.
