# Design Patterns - Introduction

So how to study these 26 design patterns without getting overwhelmed and how to remember what these design patterns do? Well, there is an easy way to study them, and that is to group them in different categories. So first of all, we have creational design patterns, then structural. And lastly, we have behavioral design patterns. So once we understand how these groups are created and what they represent, it is easy to remember where each design pattern belongs. So let's begin. First of all, we have creational design pattern. So all the design patterns that belong to this group, they deal with creating objects from classes. So they will give you solutions to common problems that you will face when you're thinking about creating objects from your existing classes. Next up, we have the structural design patterns. So these design patterns deal with how we create or compose objects so we can add another object to our existing object. And that way we can have a composition. So these design patterns deal with how to arrange our classes and objects so that we can derive a functionality out of them. And these are structural design patterns. Lastly, we have behavioral design patterns, so these design patterns deal with how classes and objects interact or communicate with each other and how we can design the interaction between groups of objects so that we can derive a functionality or we can achieve the desired goal with these objects. All right, So these are the three categories of design patterns, and we are going to study. Each of these groups are design patterns that belong to each of these groups one by one.
