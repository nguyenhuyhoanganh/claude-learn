Because these classes do allow us to build our final product in steps. So we have the append method in string builder and in each invocation of that append method, we provide only part of the final string. However, as soon as we start to look a little bit deeper into the implementation and capabilities of these classes, then this particular definition or example starts to fall apart. Because one of the requirements that is given by the definition in the Gang of Four book for builder pattern is that builder has potential to build different representations of product interface using same steps. So if you remember the UML diagram that we studied, we saw that our product can have multiple subclasses and those can be built by our builder using the same steps. Okay.
