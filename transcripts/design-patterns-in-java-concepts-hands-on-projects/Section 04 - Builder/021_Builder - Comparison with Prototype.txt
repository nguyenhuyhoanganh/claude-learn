# Builder - Comparison with Prototype

Now, in this video, we are going to look at differences between the builder design pattern and another creational pattern called as prototype. Now, we haven't yet looked at prototype design pattern. So what you can do is you can skip this video, learn about prototype design pattern and come back here or I will briefly explain what prototype design pattern is and then we will go over differences between these two design patterns. All right. So prototype is a creational design pattern. Now, what prototype does is that it uses object cloning. So if you are familiar with object class in Java, you will remember that it has a protected method called as clone. Now what clone method does is that it clones the existing object, so it will create a copy of your existing object and return it. So that is one way to create an object and prototype design pattern basically uses this clone method in order to create objects of a class. All right. So now we have learned a little bit about prototype design pattern. Let's look at the differences between these two. Inside of a builder. We will typically have a complex constructor and our builder design pattern allows us to work with that constructor. So if you remember all of our examples, the two examples that we have seen, we have always used the constructor of our product class to create the object in prototype design pattern. However, whoever is going to use our design pattern is going to call the clone method and which will create the object. So the user of prototype design pattern doesn't bother or doesn't work with the constructor of the product class. So that's one difference. Next difference, you will see that we can create Builder as a separate class. In the first example, we saw that we created a separate class for our builder and we provided all the methods that allow the user to create parts of our final object. So this makes our builder design pattern easy to work with. Legacy code. So legacy code is a code which is there for historic reasons, and you cannot change that code without incurring high cost of retesting the entire thing. So whenever you have such type of code, you can still use builder with classes from that legacy code because your builder itself is a separate class which you can write test independently. Now, when it comes to prototype, at least in Java, we implement prototype design pattern by implementing or overriding the clone method from the object class. That means we need to modify the existing code. So using prototype with legacy code may not be always feasible or possible. All right. So these are the differences between builder and prototype.
