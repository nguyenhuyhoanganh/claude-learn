Instructor: Git tags. In this lecture, we are going to see how we can use Git tags for semantic versioning. First, let's understand what is a semantic versioning. Now, almost every software today uses semantic versioning. If you take example of a browser, Brave, take a look here you have version 1.71.114. Okay, now get 3.7.6. Visual Studio 1.91.1. See the similarity? These are three numbers, right? Separated by dot. Okay, let's understand this. So this is the format. You have Major.Minor.Patch. The patch indicates that there is a bug fix. Simple slight improvement from the previous version. Minor number indicates that there is new feature added or some improvements that was done in the software. Major version indicates major changes and it will be backward incompatible. Like let's say a vprofile project used to support JDK 11 and then we upgraded it to JDK 17.
