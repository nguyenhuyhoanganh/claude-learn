So first, let's understand what is GitOps and then we'll understand about this project. So GitOps means Git repository will be the single source of truth for any operational changes. So Git here represents that everything, all your infrastructure information, your application information, everything will be stored into the Git repository. And there will be some pipeline, some controllers that will make sure that your infrastructure, your application, everything is as per the code in the Git repository. Why this is such a big buzz in today's DevOps world? What we have seen so far, when we do manual deployment, it's time consuming and error prone, and it's hard to audit and track. So if I made any change with my automation pipeline or with my automation code and someone is also making manual changes, then it becomes difficult to track who did what. And also, it's not scalable if you have a big team, if your team is growing. Because of this, you cannot replicate everything into the next environment. Like if you have GitOps, you can have dev environment as same as the production environment, at least in terms of configuration. In terms of infrastructure size, it will be definitely different. But there are many more benefits that we get with this GitOps platform. So GitOps platform means everything is automated, everything is auditable because everything is in Git repository, we'll know who committed.
