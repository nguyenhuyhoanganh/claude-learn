Instructor: Hello, and welcome to the AWS CI/CD project. In this project, we are going to use AWS services to fetch your code, build it, and deploy it on AWS Elastic Beanstalk. We have previously seen Jenkins CI/CD project or Jenkins CI/CD pipeline. Same way, we are going to create a CodePipeline, but we are going to use AWS services for that. So to complete this project, first, we are going to create AWS Elastic Beanstalk environment. So we are going to build the code and deploy it on Beanstalk environment. Then we will be creating AWS RDS. Because our Vprofile application needs database connectivity for that, we will be using AWS RDS. For the source code, we will be using Bitbucket repositories. We have previously used a GitHub repository. In this project, we will be using Bitbucket repository, and I will talk about this in a moment. So Vprofile source code lives on GitHub repository. We are going to migrate it from GitHub to Bitbucket. Then we are going to use AWS CodeBuild service, which is going to fetch the source code from Bitbucket and deploy it on Beanstalk, or course, after building the artifact. We are going to use AWS CodePipeline service to connect all these services together. And finally, we are going to test our CI/CD pipeline by doing a git commit and pushing the changes to our Bitbucket repository, which will trigger our pipeline. Now this is the first time we are using Bitbucket repository, so let me quickly talk about it. Bitbucket is from Atlassian, and it's a very popular name among developers and also among DevOps engineers. So Bitbucket is from Atlassian, and you can think Bitbucket as GitHub. So all kind of Git solution it provides. There are many organization that uses a Atlassian services. Along with that, they use Bitbucket repository. So there is a high chance when you work in real time, you will be using Bitbucket repositories or Git repositories on Bitbucket. Also, along with the Git repositories, it provides the complete CI/CD pipeline, various services in that pipeline. So before we begin, let's take a look at the architectural diagram. So our pipeline will be like this. We will have Git repositories on a Bitbucket, which we will migrate from GitHub to Bitbucket, AWS CodeBuild service, which will act like Jenkins build, which will fetch our source code from Bitbucket Git repositories, build our source code into an artifact, and then deploy it on Beanstalk. Now there's no default option to deploy it on Beanstalk. We will be using AWS CodePipeline service to connect all these pieces together. So CodePipeline will detect any new commit on our Bitbucket repository, fetch the source code, trigger the CodeBuild project, which will build the source code into an artifact, and then deploy it on AWS Elastic Beanstalk. Now, as you already know, our Vprofile application needs database connectivity, so we will be having Amazon RDS, which will have schemas, tables, for our Vprofile application.
