# Why Monitoring is Essential for DevOps

So why should DevOps bother about monitoring? There will be separate monitoring team. SOC team or NOC team that will do 24 by seven monitoring. And so far, mostly the administration team is responsible in setting up monitoring. NOC team will do the monitoring. And based on the information NOC team can escalate the issue. But in today's time in modern applications, in SRE, they say you build it, you run it. Now DevOps own the deployment and reliability. So understanding these metrics and logs will help DevOps communicate better with the administration team with the NOC team SoC team. Automate these deployment process of monitoring. Monitoring agents needs to be set up. Or the exporters we call it in Prometheus needs to be set up. So due to that, it becomes essential for DevOps engineers to understand these concepts. The NOC teams watch with DevOps team will decide what is worth watching. And now we can automate the process of creating these dashboards, alerts, checks, metrics along with the infrastructure setup. So that's one more very good reason why DevOps should learn monitoring. If something goes wrong, usually the NOC team will escalate to the administration team and now the DevOps team. So DevOps or SRE can find out the root cause, which will speed up the recovery process with monitoring and observability. We can collect a lot of information and based on this data, we can decide about auto scaling, different kinds of rollout, CI, CD deployments. When we do, the feedback will come very quickly. So understanding these monitoring tools, observability tools will help DevOps make right decisions when to scale, when to scale down or scale out, etc.. And last but not the least, DevOps is all about collaboration and continuous improvement. So this will connect dev team, ops team and NOC team together, which will lead to better performance of your application and of course the stability. So there are several good reasons for DevOps to learn monitoring and observability. But doing 24 by seven monitoring will be the job of NOC SoC team. Now finally there. Disclaimer this is not a full fledged monitoring and observability course, but as a DevOps engineer, you will be equipped with enough knowledge to work with monitoring NOC team and even developers.
