# Introduction

Hello and welcome. In this section of the course we are going to learn GitLab, a complete CI CD solution stack. GitLab is a complete DevSecOps platform. You get everything at one place. You have the source code management git repositories. You have CI, CD. You can implement security and complete project management. You get both model. You can use SaaS. You can just subscribe, login to gitlab.com, subscribe and start using it. Or you can also host your own GitLab environment. The three options available you have free tier. You can go for Premium and ultimate tiers. In this lecture we will be using free tier. So first you have the source code management, the complete git repository setup. Just like any other git repository, you get option of hosting your repository. You have branching workflows updating code in branches through merge request. Well, that's the pull request in GitLab. It's called a merge request. With approvals, of course, you can use web based IDE. You do not prefer VSCode like your own code editor. You can use web IDE. Also, you have wikis for maintaining documentation and all the amazing features of the git repository you get in GitLab. So this is where we start. And then we need continuous integration and deployment pipelines. So you can set YAML based pipelines. Easy to read, easy to write. Looks something like this. Complete YAML based pipeline for managing multiple stages in your pipeline, multi-stage jobs and pipeline visualization. You can see all your pipeline and different jobs in your pipeline at one place. With all the logs and all the information, you can use inbuilt runners from GitLab, or you can add your own runner. That means you can decide where your job should run. You can choose external servers like EC2 instance, physical machines or VM's hosted in your own environment. You can select a runner for your project like I have. An EC2 instance is added over here. I can enable and disable it or like that add any other runner of my choice. Auto DevOps templates are available which you can use ready made and also integration with many other platforms like Kubernetes. You can use GitLab agents that safely connects to your Kubernetes cluster. You can also integrate with Google Cloud. Also, you can keep an eye on your Terraform states from a security point of view and also compliance. You get a code analysis, static code analysis, or dynamic testing. So not only code scanning, container scanning. If you're building Docker images, license checks so you or your organization doesn't get into legal issues. So in your pipeline you can integrate security capabilities which is mandatory in DevSecOps pipelines. Secret detection so you don't accidentally expose your credentials to git repositories or any other public platforms. We have audit logs and also you can set compliance pipelines. The outputs of your pipelines are usually packages or artifacts. If you're publishing Docker images, GitLab gives you container registry where you can host your Docker images. If you need a package, registries for example for Maven or npm, you can use package registry provided by GitLab or any other artifact that you want to store, like reports. You can mention that in your pipeline and the job will store your artifacts as well. Monitoring and analytics. You get complete analysis of your pipeline. You get a dashboard. You can look at the trend. How many pipelines passed or failed. We have code quality reports. GitLab can integrate with your monitoring tool like Prometheus and monitor health of your application like CPU, memory, disk also integrated with sentry so you can do error tracking. Also audit events. Who did what? For complete project and team management, you can develop dashboards which has issues, set milestones and roadmaps. On top of that, you have service desk for customer support. If all that is not enough, GitLab also has various integrations with cloud platforms with communication or notification systems like slack tools integration like JIRA, Trello boards, or even Jenkins. And like any other web based platform, you have APIs and webhooks as well. So in conclusion, GitLab provides you everything that unifies development, security, and operations for teams. You have collaboration, automation, compliance and I have personally used it for startups and even enterprises. So let's see GitLab in action from the next lecture.
