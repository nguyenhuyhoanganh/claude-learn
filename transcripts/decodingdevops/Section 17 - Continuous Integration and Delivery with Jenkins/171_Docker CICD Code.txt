# Docker CICD Code

All right, let's take a look at the code that we will use to deploy our container on ECS. First, we are setting here two variables, cluster and service. Cluster will be the ECS cluster name that we will set up in the next lecture. And service is the service that we create in the ECS cluster. Service in ECS clusters gives complete management of your container. It gives load balancer, health check monitoring, logging, all in one. So in the next lecture we are going to create ECS cluster. And in the cluster, we are going to create the ECS service. Also, the service is going to fetch the image from ECR repository from this path and run it on the ECS cluster. So to deploy a container, the stage is pretty simple. The rest of the code is exactly same as we have seen previously. We have a stage deployed to ECS. In this, we have the step here. We are using a plugin, Pipeline AWS Steps. Now this plugin gives us a feature where we can mention details like credentials, awscreds. This is the credential that we have saved in Jenkins and the region. So it's going to set this region and credential. And with that, you can execute your command. So this is a shell command. So on the shell, it is going to export the credentials, the region information, and execute this shell command. This is an AWS CLI command. This command is going to check for the latest image available in the repository, fetch that container image, and execute your container on the ECS cluster. So we see aws ecs update-service -cluster, cluster name. This is the variable that we have set. -service, the service name, again, the variable that we have set, and we force for a new deployment. When we do this, it's going to go and check for the latest image available in the ECR and run it on this cluster under this service. So once again, we are going to create ECS cluster in the next lecture and ECS service inside that. And after that, we are going to install this plugin in Jenkins. So join me in the next one.
