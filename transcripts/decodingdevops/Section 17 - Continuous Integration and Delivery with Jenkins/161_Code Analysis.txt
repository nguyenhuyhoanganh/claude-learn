# Code Analysis

Welcome. In this lecture we are going to talk about code analysis. Now you have already enjoyed writing pipeline as a code where we fetch the code from GitHub, build it by using Maven, and execute unit test cases. The next step is to do code analysis. But let's understand first, what is code analysis. Code analysis is a test that helps you follow the best practices while writing the code. It's a process of examining the source code to identify any potential issues and improve your code quality. Let's understand why we do code analysis. As I already said, it is to improve the code quality and also to detect security vulnerabilities. If developers do not follow best practices while writing the code, they might develop security vulnerabilities in the code that will be used by hackers to exploit the code, the application. So, code analysis in a very early stage detects these vulnerabilities and make suggestions. While writing the code, developers will be chasing deadlines. Developers will take shortcut while writing the code, which will later create issues. So, code analysis will help catch this in the very early stage, and it definitely helps the team follow the best practices and the best coding standards, consistent across the code. Most importantly, it saves time by catching these issues earlier before it goes to production. And since it's automated testing, it helps in our DevOps workflow and our CI/CD pipeline. There are many code analysis tools available out there in the market. Some of the name I'm giving over here, the popular choices. One is SonarQube that we'll be using in this lecture.
