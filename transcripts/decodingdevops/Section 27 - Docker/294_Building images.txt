# Building images

Now we will learn how to build images, docker images. So far we have taken official docker images from Docker Hub and we ran them. But as per our requirements, we need to make changes so we can take an official image, make some changes and build our own customized image. Like we create AMI on AWS. To build docker images, you need to write docker file B capital. We provide instructions in this file of how to build the image, what should it contain, what packages it should install, what volume to export. And then we run docker build command which will create an image from the docker file. Just like we have palm XML for Maven which builds the artifact. Similarly, here we'll have docker file that is going to build an image for us. So what contains in this docker file are docker file instructions, some set of commands, pretty simple, no programming, just instruction and their values. Like from you can specify the base image like Ubuntu or MySQL. An official image will be always good for this. So you take a base image from Docker Hub and then on that you do customization. You can put labels. Labels are like key value pairs. Just like we have tags on AWS. Like you can give project name, author name. RUN RUN, This is the instruction that is going to make changes. Like we want to install a package or maybe create a directory or a file, whatever command we want to execute to make changes that we mentioned through run instruction. ADD or COPY you have two instructions that will do almost similar thing. It will copy or add a file to your image while you're building it, like a configuration file or an archive. The difference between add and copy is copy will just take the file and dump it. Add instruction you can provide a link, an archive so it can download from a link and then put it in your image, or it can even unarchive an archive file. CMD instruction is to set what binary will get executed when you say docker run. Entrypoint which is similar to CMD but will have higher priority if you mention them both together. You can mention command in the entry point and argument to that command by using CMD. VOLUME Which volume would you like to export? Like we have seen for MySQL, it's /var/lib/mysql. EXPOSE So mentioning your binary in the running in CMD instruction CMD or entry point, a binary runs and creates a process. That process could be binding to a port number. Let's say for nginx we have port 80 or MySQL we have 3306. So whatever port the process is binding to, you have to give the same port in the expose. Then you can do port mapping when you do docker run. Few other instructions like ENV to set environment variable, USER to specify what user will be running the process. WORKDIR which is going to set a working directory. Like when you do docker exec and you mentioned any command, that command will be running from the work directory. ARG You can specify variables that the user can pass at the build time. On build instruction is useful when you are going to use this image as the base image. When you are building some other images, you can specify any instruction, any command on that, and it will run that whenever this image is used as the base image. So let's learn by an example.
