# Section 7 Intro

We talked about Logstash earlier in the course, as a way to publish server logs into an elastic search cluster as they come in. But this architecture has its limitations, it's easy for logs to actually get backed up without some intermediate layer in place. That's where Beats, another part of the elastic stack comes in. It offers a very lightweight utility that resides on your web servers to publish log data to a separate log stash cluster, which then serves as a buffer to your elastic search cluster, and it's easy to set up and use; it'll make a lot more sense when we just use it together. So let's talk about how it works and just dive in.
