# Index Lifecycle Management

In addition to Alias rotation, Elasticsearch has a feature called Index Lifecycle Management. It allows you to control how indices are handled as they age by assigning lifecycle policies at each stage of the lifecycle. And instead of managing indices on a fixed rotation schedule, you can choose to trigger these events based on things like shard size and performance instead. Now, there are five stages these days. First is hot, where the index is being actively updated and queried, warm where an index is still being queried but no longer updated. So it could still be read only at this stage and cold. At this stage the index is no longer updated and it's only queried infrequently. At that stage you can get away with those queries being slower, maybe using less expensive hardware to save money. And new is the one called frozen, which is like cold, but updated even less frequently and it's even slower in that stage. Finally, there's delete and eventually you probably want to get rid of this index entirely. You might want to delete it entirely for compliance reasons or legal reasons, whatever it might be. Now you do find policies for each stage which let you specify things like the size or age at which you want to roll over to a new index when the index can be deleted and stuff like that. Here's an example of a lifecycle policy. If you examine it, you'll see it simply defines two stages hot and delete. Once an index reaches 50 gigabytes in size or 30 days in age, it's rolled over to the delete stage where it will remain for an additional 90 days before being deleted. Now you need to apply this policy to actual indices. And to do that, we use an index template like this. In addition to configuring the charts and replicas, we're also configuring the lifecycle policy using index start lifecycle name and also defining the alias used for the rollover action defined in that policy with index start lifecycle dot rollover underscore alias. That alias still needs to be set up as we showed earlier when we talked about alias rotation. You've seen how different indices can be used for a variety of administrative functions and to make your data scalable and to move it through different stages of its lifecycle. These are all important concerns in a production setting.
