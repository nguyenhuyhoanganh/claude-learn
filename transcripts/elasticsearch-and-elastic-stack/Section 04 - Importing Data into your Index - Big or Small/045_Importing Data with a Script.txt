# Importing Data with a Script

An often overlooked topic is importing data into Elasticsearch. And the good news is that there is a very rich ecosystem surrounding Elasticsearch these days. So no matter where your data may live, whether it's some big static repository or weblog streaming in real time, odds are there's a pretty easy way of getting it into Elasticsearch. There's a variety of ways of doing it, and we'll cover a lot of them in this section. You can just write a standalone script where you actually write some code to do whatever you want to suck data in from wherever it may live, and feed it into Elasticsearch, either through JSON requests or through client APIs. Or you can use off the shelf components like Logstash and beats. These are both parts of the larger Elastic Stack, and these have a wide variety of importers and exporters built in that can talk to pretty much any system you can dream up. It's just a matter of configuring them. And oftentimes you can just install Logstash or the beats framework and just configure it and go. You don't need any code at all. If you're dealing with Amazon Web Services, they also have some systems for streaming data in from other AWS components. For example, AWS Lambda service can be used to funnel data from one system into Elasticsearch on AWS. And you can also use Kinesis Firehose if you have data being shuffled around by AWS Kinesis and systems beyond that, such as Kafka and Spark, which are larger systems that might be shuffling your large data sets around on your clusters already probably have an Elasticsearch integration and add ons built in. And we'll look at examples of doing that with Kafka and Spark in this section.
