# Elasticsearch and Apache Spark, Part 1

So while Elasticsearch is a hot big data technology, so is Apache Spark, perhaps even more so. And it turns out you can integrate the two together. So let's explore that. What is Apache Spark? Well, basically it's the successor to MapReduce for doing large scale data processing on a cluster. Often you see it running on a Hadoop cluster, but it can also run on its own standalone cluster as well. And it's just made for doing distributed processing and transformation of data, um, you know, horizontally distributed across a cluster. But the thing is, you can't just drag and drop stuff around with Apache Spark. You have to actually write code for it. And those are called Spark Driver scripts. You can write that in Java, Scala, Python or R, Scala being the preferred language because that's the language that spark itself is written in. And in addition to just allowing you to transform data, it also has a bunch of add on packages sitting on top of it. There's spark SQL, which allows you to issue SQL queries to your spark data, which might be distributed across your entire cluster. There is spark streaming, which kind of like Kafka, allows you to process streams of data in continuously and analyze that data as it's being fed into your cluster. It also has machine learning capabilities, which are very popular, obviously, and it also offers graph processing. And we're not talking about charts and graphs and spreadsheets. We're talking about graph data structures like social networks and things like that and exploring the properties of those networks. I should mention that Apache Flink is another similar project that's nipping at Apache Spark's heels these days, and that can also integrate with Elasticsearch. But for now, spark is still the winner. So we're just going to cover spark in this lecture. Let's look at an example here.
