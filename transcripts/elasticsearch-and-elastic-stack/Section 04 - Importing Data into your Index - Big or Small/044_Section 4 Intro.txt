# Section 4 Intro

Earlier, we covered importing documents individually and in bulk, using elastic search's REST API, but in the real world, that's not always a practical approach. Your data may not be in Json format already, or it may be stored in some large distributed system, or maybe you need to import streaming data into elastic search as it's generated. The topic of importing data into elastic search is often overlooked, but that's what this whole next section is about. We're going to write scripts to import raw unstructured data into a new index, we'll use logstash to import web log data into a new index as it's generated. We'll import data directly from a sequel database and from data stored on Amazon's S3 distributed storage system. Finally, I'll show you how to integrate other systems, including Kafka, Hadoop and Spark with elastic search. Wherever your data is coming from, odds are, there's a simple way to get it into your elastic search cluster. Let me show you how it works.
