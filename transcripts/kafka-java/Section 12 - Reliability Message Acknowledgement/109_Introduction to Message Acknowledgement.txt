# Introduction to Message Acknowledgement

In this section, let's talk about message acknowledgement. This is also called offset commit. In the Kafka fundamental section, we had said that Kafka maintains some kind of ledger to track offsets. That is, Kafka tracks what messages have been delivered so far for each consumer group. Let's quickly refresh that. When we describe a consumer group, we can see this output. Here we see the topic, the partition, and the log end offset, which shows the last offset in the partition. Then the current offset, which shows the last message the consumer group has seen so far. And the lag, which shows how many messages Kafka still has to deliver. If the lag is zero, it means the consumer has consumed everything. There is nothing more for Kafka to deliver from the broker side. Now we might be wondering, how does Kafka actually track this? When does Kafka update this current offset? Is it when Kafka delivers the message to the consumer? Let's say this consumer asks for messages. The broker sends some messages. Now as soon as it sends the message, does it go and immediately update the current offset? Actually no. Under the hood, something interesting is happening. Kafka is a streaming platform. It has to reliably deliver the messages. Kafka needs to make sure that the consumer has seen all the messages, processed all the messages. So when a consumer asks Kafka for messages, Kafka will be delivering the messages like this, but the Kafka does not immediately, okay, I have delivered the messages. Let me update the offset. It does not do like that. Instead, Kafka will be expecting an acknowledgement from the consumer. The consumer has to say, yes, Kafka, I have seen these messages. I have processed the messages. So the consumer must say this to Kafka. Only then Kafka will be updating the offset. Otherwise it will not be updating the offset. So the consumer application must acknowledge the messages. So this is why people use these two terms interchangeably, acknowledgement and offset commit. This acknowledgement is, it's an application level concept. This is what normally we developers say offset commit is the Kafka term, it's a Kafka operation behind it.
