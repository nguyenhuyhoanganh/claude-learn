# Section Summary & Key Takeaways

Let's quickly summarize whatever we discussed in this section. Kafka tracks the current offset for every partition and for every consumer group. That is how Kafka knows which messages are already processed and which ones are still pending. To update that offset, Kafka expects a confirmation from the consumer. The consumer application has to tell the broker that yes, I have seen these messages. You can move the offset forward. Only then Kafka commits the offset. Now in our case, we did not really do any of this manually. That is because the Spring Cloud Stream handles the acknowledgement for us under the hood. This is very similar to how the Spring Data JPA manages database transactions. We focus on the business logic and the framework takes care of committing things at the right time. And this is the recommended approach. If you are using Spring, let the framework manage the acknowledgements for you. However, if you ever feel that you need more control, then Spring gives us the option to acknowledge the messages manually. To do that, our consumer Bean needs to accept the message of type T. Then in the configuration, we enable the manual acknowledgement board. Once we do that, Spring attaches an acknowledgement object to the message headers. Then we can access the message headers to get the acknowledgement object like this. We can process the message and if everything goes well, we explicitly acknowledge the message. If something goes wrong, we can use NAC to retry the message after a delay. NAC is not a Kafka concept. It's a framework feature provided by Spring for controlled retrace and error handling when you use the manual acknowledgement. However, there are better techniques. We are going to discuss as part of the upcoming sections. So no worries. We have a dedicated section for error handling. Then finally, when we consume messages in the batch mode, the acknowledgement works a bit differently. We do not get one acknowledgement per message. Instead, we get a single acknowledgement object for the entire batch.
