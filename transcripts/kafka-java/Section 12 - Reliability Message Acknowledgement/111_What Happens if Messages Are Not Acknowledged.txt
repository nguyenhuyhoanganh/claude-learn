# What Happens if Messages Are Not Acknowledged?

This will be a quick lecture. Let's consider this scenario and this is very important. Let's say this is our payment processor application. It asks Kafka broker for messages. Kafka sends four messages. So our application processed the payments for those four customers. Great. Now, when this application is about to send the acknowledgement to Kafka, exactly at that moment it crashed. So Kafka did not receive any acknowledgement. So what will happen? The application restarts. It joins the Kafka broker again. It asks for messages. So will Kafka be re-delivering those four messages again? Yes. In that case, this application, will it not process the same four messages again? Yes, if you are not careful. This is one of the challenges in an event-driven architecture. Remember, this is not anything to do with Kafka. Kafka cannot help with this. This is our application design problem. It has to be fixed using some patterns and best practices. I am going to cover that later in the best practices section. But this is something you should be aware of.
