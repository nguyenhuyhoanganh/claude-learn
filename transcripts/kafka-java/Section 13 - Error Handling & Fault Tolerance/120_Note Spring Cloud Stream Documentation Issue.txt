# Note: Spring Cloud Stream Documentation Issue

This will be a very quick lecture to share this information. At this time of recording this lecture, the maximum attempts, the behavior does not seem to work as they describe in the documentation. For example, when you say maximum attempts 3, that includes the very first attempt. That's what the description says here. That is, when you say 3, first you will have one message delivery attempt. Then in case of failure, you should be having 2 retry attempts. That is how I interpret this when I read the documentation. However, when you give 3 like this, right, when you see the actual demo, first you have one delivery attempt. Then after that, you have 3 retry attempts. So this is how currently it works. So one of them could be wrong. So this is something I want you to be aware of.
