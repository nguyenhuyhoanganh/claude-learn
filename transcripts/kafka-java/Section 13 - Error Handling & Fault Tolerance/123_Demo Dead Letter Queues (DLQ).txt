# Demo: Dead Letter Queues (DLQ)

In this lecture, let's talk about DLQ, Dead Letter Queue or Dead Letter Topic. We can retry an exception few times, but we cannot retry forever. At some point, we will want to go to next message. A DLQ is simply a topic where messages go when they cannot be processed successfully after all the retry attempts are exhausted. At this point, retrying no longer helps. The problem could be invalid data, a corrupt message or it could be an issue with our application that needs manual intervention. So instead of blocking the consumer or losing the message, we acknowledge it and publish it to a separate topic, the DLQ, along with error details. So that we can inspect the messages later from this topic and once the issue is fixed, we can even reprocess these messages. As usual, we do not have to write any special code for this.
