This will be a quick lecture. As I had mentioned, as part of the derived binding names, we will be using the index as zero throughout this course. But when do we use the index as one? What is the use case for it? That is what we are going to discuss. Spring team seems to have got the inspiration from reactive programming for this use case and it's supported only via reactive programming at this point. In reactive programming, we can have two flexes representing the stream of items like this driver stream, passenger stream. Using the zip operator, we can merge these two, that is take one item from this flex and one item from this flex to produce another object. For example, take one driver and take one passenger to create the trip object. Looks nice, right? Similarly, imagine two topics, driver topic and passenger topic. Now, if you are comfortable with the reactive programming, we can expose a bean like this, which accepts a tuple two. So for the consumer bean, the consumer in zero will be accepting driver topic. Consumer in one will be accepting the passenger topic items. So if you expose the bean like this with a tuple two as the input type. So here, spring cloud stream will be giving you two flexes. The flex of driver and the flex of passenger. Using that, you can get the driver and the passengers and you can merge, you can do whatever you want to do. So this is the use case the spring is targeting. Please do note that it works only with the reactive programming as of now. For most of the real world use cases, I think we do not need this. So let's focus on single input functions.
