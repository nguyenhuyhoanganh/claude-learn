# Launching the Kafka Container

In the previous lecture, I would have shared the repository. If you clone the project, you would be seeing many directories. For now, let's focus on the 0.1 Kafka setup. I have created one Docker Compose YAML to start Kafka in our local. As you see, it's very simple. We use Apache Kafka image. And in the video lectures, you might be seeing this version. But as I had mentioned, I update the project periodically. So you might be seeing a different version. So please use that whatever I provide. Then I provide the working directory here because this path in the container, this path contains the command line tools we will be using actually. So when we access the container, we will be landing in this path. So this is why I provide the working directory. Then Kafka listens on port 9092. So I do the port mapping. Very simple. Please open the terminal, go to the path 0.1 Kafka setup where you have the Docker Compose YAML. Then let's do Docker Compose up. If you want, you can also use the minus D option to run in the detached mode. That's up to you. So I'm going to use this Docker Compose up, hit enter. That's it. Kafka server started. If you scroll up and see, whenever Kafka server starts, it will be printing all the properties it uses. It's a lot. We can, of course, override these. We will be talking about all these things later, but I just wanted to show you this.
