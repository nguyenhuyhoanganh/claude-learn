# Kafka Docker Images

In this section, our goal is very simple. We are going to set up Kafka in our local using Docker Compose for our learning, local development and testing, nothing else. At this stage, I might not be explaining every command or every property you see. For example, you might be wondering, why are we running this command? What does this property do, et cetera. Kafka has many moving parts and all the concepts need to be introduced one by one in a specific order. If I try to explain everything upfront, it will only create confusion. So please be patient. Let's take this slowly. We will learn a lot as we make progress in this course. The Apache Kafka project provides two official Docker images and it's important to understand the difference. The first one is Apache Kafka. This is the standard JVM based image. It includes all the Kafka command line tools, which we will be using in this course. This image is a recommended for a general use, including the production setups. The second image is Apache Kafka native. This one uses GraalVM for faster startup. It's experimental and it does not include the command line tools. It's mostly used for integration testing in the CACD pipelines. For this course, we will be mostly using Apache Kafka, the standard image for our learning. Later when we write integration test, that time we will be using this image. Now a quick word about the Kafka version we will use. Kafka team releases new versions every now and then. And sometimes these are simple bug fixes and the internal architectural changes, et cetera. But the core concepts, they are stable. They have been the same since Kafka version one. I'm going to share a GitHub repository in the next lecture. That repository will contain all the source code, whatever I do in this course. I will be updating the GitHub repository periodically, like every three to six months to keep everything current. And I will be updating the version as well. So please use the version I provide.
