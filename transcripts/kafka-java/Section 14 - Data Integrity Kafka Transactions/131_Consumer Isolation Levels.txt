# Consumer Isolation Levels

Let's talk about one small but very important setting on the consumer side, isolation level. This isolation level simply decides what a consumer is allowed to see. We have two options, read committed and read uncommitted. By default, the Kafka consumer use the read uncommitted option. That means the consumer reads everything from a Kafka topic. It does not matter it's committed or the transaction is aborted, it can see all the records. When we use the read committed option, the consumer will see records only from the committed transactions. If the transaction is aborted, those records are not visible to this consumer. To better understand this, let's assume we have one Kafka topic and we have messages like this. There are three transactions, transaction 1, transaction 2 and transaction 3. We have messages like M1, M2, M3, M4 and M5. They all belong to different transactions like this. The transaction 1 is committed, transaction 2 is aborted and the transaction 3 is still in progress. We have two consumer applications subscribing to the topic with different isolation levels. This uses read uncommitted, this uses read committed option. The read uncommitted consumer will be seeing all the messages M1, M2, M3, M4 and M5. The read committed consumer will see only M1 and M3. They belong to the transaction 1 and it's committed. So because of that, those messages will be delivered to this consumer. This M2 and M4 cannot be delivered to this consumer because it has been aborted. And M5, we do not know the status yet, it's still in progress. So in order for the broker to deliver this message to this consumer, we need the commit marker for transaction 3. So hope it's clear. The read uncommitted is the default option is mainly because of backward compatibility. Kafka transaction was not there from the beginning. So they added this feature a few years ago. So the read uncommitted is the default option. You can use read committed if your producer or processor is producing records in the transactional mode. Otherwise, you can just go with the default option.
