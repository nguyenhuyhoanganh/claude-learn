# Section Summary & Key Takeaways

This will be a quick lecture to summarize whatever we had discussed in this section. A processor other than doing the transformation, it can also act as a router. Event routing is a process of directing an event or a message to one or more destinations based on some rules. If we route a message based on the message content, then it's called content-based routing. But if there are other external factors like service availability or feature flags, etc., then it's called dynamic routing. These are all concepts. For the routing implementation, Spring Cloud Stream provides two simple options. One is to use the stream bridge, consume the event first, then have your business logic to decide where to send the output event. Another option is to use the message header.
