# Content-Based Routing

In this section, let's talk about event routing. The main goal of this section is to understand what routing is and discuss event routing patterns. In the previous section, this is what we did. We developed a processor application which transformed the data. Order to payment, order to notification and so on. A processor does not just transform data, it can also act as a router. Let's define what routing is. It's a process of directing an event to one or more destinations based on some rules, some conditions. To understand this better, let's take this example. Our producer produces order events. Our processor consumes these events. Now, we have two types of products. Digital products, something like e-books. Then we have physical products. The digital and physical products are handled very differently in the real life. A digital product can be delivered instantly. For example, sending a download link or granting access to the user. But the physical product requires shipping, creating a shipment, assigning a carrier, tracking, delivery, etc. So, we have two different types of consumer to handle the delivery. Digital delivery consumer, physical delivery consumer. The digital delivery consumer is one microservice which knows how to generate a secured download URL and send that to customer email, etc. The physical delivery consumer is another microservice which knows how to create a shipping label assigned to carriers like FedEx for delivery, etc. So, the processor checks the incoming order object. Depending on the product type, it creates a corresponding delivery object for the order. Then it sends to appropriate topics. If it's a digital product, then it will send that to digital delivery topic so that this consumer can handle that. If it's a physical product, then it will be sending that to this topic, physical delivery topic, so that this consumer can handle that. This is what we call event routing. We have two different types of event routing. One is content-based routing. The other one is dynamic routing. Whatever we just discussed, right? Routing depending on the product type is an example of content-based routing. That is, you route the messages based on the content of the message. So, you receive the order object, right? The processor checks the order object. So, here depending on the product type, it routes. So, it's content-based routing. First, let's see how to implement this. Then we can talk about the dynamic routing later.
