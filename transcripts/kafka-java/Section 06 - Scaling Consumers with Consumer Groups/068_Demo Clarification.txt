# Demo Clarification

Now, I am sure some of you will be having some questions or concerns that I simply just printed messages on the console and called this demo. I tried to prove something with these numbers, but it does not really prove anything. I can guess this question. In the real life, consumers will be processing the messages, will be making some database calls, message processing will take some time, instances could crash while processing the messages, etc. Right? So you might argue that this demo is not accounting those scenarios. Okay. In that case, we are talking from slightly different perspectives. That is, please check the goal of this section. The whole idea of this demo is to understand how well Kafka works. It is not about our application. This demo is about Kafka, how Kafka distributes the partitions, how elegantly it handles when consumers join and leave a group, etc. Your concern is mostly related to application error handling scenarios, which we have not yet discussed. We will be covering that as part of upcoming sections. But again, this is about Kafka to show that how well it distributes the partitions among available consumers. Hope you get the idea.
