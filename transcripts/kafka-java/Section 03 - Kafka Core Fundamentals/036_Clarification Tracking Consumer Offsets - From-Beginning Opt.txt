# Clarification: Tracking Consumer Offsets - From-Beginning Option

This will be a quick lecture and I would like to explicitly mention one important information even though I kind of said that in the previous lecture, so maybe I am repeating this, sorry about that. Ok, so in the previous demo when these consumers subscribe to this topic for the very first time they did not specify from beginning. Because of this reason even though we have, we had some messages in those partitions without delivering those messages, Kafka started tracking the current offset as 3 and 2 but it did not deliver the messages because these consumers were not interested, right. If these consumers, if they had specified from beginning, it would have delivered those messages, ok take 3 messages, take 2 messages, so it would have delivered like this and it would have tracked 3 and 2. So the from beginning or latest, it works only for the very first time when it starts creating this ledger or when it starts tracking for the very first time, initially it would, it will not have this entry. So only for the very first time in order to put some number it will ask, hey you want from the beginning or you want the latest. So based on that it will deliver the messages or it will not deliver the messages so that it can update the current offset. Once it started tracking the current offset for the consumer group, after that this will not have any effect. If these consumers join again by saying from beginning, Kafka will not deliver. So it will always deliver based on this offset only.
