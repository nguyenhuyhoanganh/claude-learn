# Log Retention Policies

This will be another quick lecture, Kafka stores data on disk and consumers can read that data later. So the natural question is, how long does Kafka keep the data? Will it keep it forever? Can I ask my Kafka server to deliver the messages that were produced one year ago? Okay, so the answer depends on the log retention policy. These are just simple broker level properties configured in the server.properties file. By default, Kafka keeps data for 168 hours, which is seven days. It can be controlled in two ways, based on time like this, or based on size like this. So here by log, we mean the messages in high level. So Kafka applies both rules. So whichever comes first, if one of the condition is met, then the old data will be deleted. There is also another property, it's a, there is a background cleanup interval check. Kafka periodically checks the messages and removes the data that is older than the configured time or exceeds this size limit.
