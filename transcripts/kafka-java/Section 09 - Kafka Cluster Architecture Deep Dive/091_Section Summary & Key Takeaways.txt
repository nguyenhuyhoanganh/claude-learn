# Section Summary & Key Takeaways

Let's quickly summarize whatever we did in this section. A Kafka cluster does not give the high availability and horizontal scalability automatically. The multiple brokers in the Kafka cluster provide only the capacity. Only the topic partitions provide scalability and the replication provides availability. We were able to set up a Kafka cluster with the Docker containers. We were able to create a topic with the multiple partitions and a replication factor so that Kafka could elect multiple leaders for each partition. We were able to see the followers for our topic partitions to replicate the data for high availability. We also noticed that how the bootstrap server was able to give other server information to our application. Once the application is connected to the cluster via bootstrap server, then even when the bootstrap server goes down, our application works just fine. In the cluster, nodes can go up and down, but our application was able to produce and consume messages just fine.
