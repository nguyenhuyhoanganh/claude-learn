# Networking: Understanding Kafka Listeners (Internal vs. External)

We are going to set up a cluster in the next few lectures. I want to explain what is going on under the hood in a very high level. Please feel free to skip if you find this boring. In a Kafka cluster, we can see three different types of communication. One is the controllers talking among themselves, we call that control plane. Then the brokers talking among themselves, we call that data plane. For example, brokers replicating the data, that is data plane. Then our application might want to talk to the broker for producing and consuming data. So we have three different types of communication. Among these three, the controller and the brokers communication happen among themselves within the cluster. So the outside world does not have to worry about this actually. By outside world, I mean our application might not be interested in this internal communication. So this communication happens within the cluster or within the subnet, where these are all running. To support all these different communications, a Kafka server can listen on multiple ports. These are all called listeners. For example, we might be opening a port for the controller communication, another port for data plane communication, another port for external communication and so on. Now let's imagine this is our producer application and our producer application is using this broker as the bootstrap server. Now our producer is producing order events, let's imagine like that. But this node is the leader for order events topic. Now how can this bootstrap server inform our application that how to reach this broker? Because it could be using different port for external communication, right? Each and every broker in the cluster has to provide their information how to reach them like this. For example, this broker says, this is my private IP address for the internal communication, the data plane communication. This is my external IP or public IP for external communication. Similarly, every broker has to share their information like this. By doing this way, this broker can inform our application that, hey, if you want to produce order events, this broker has the order events and this is its public IP address or external IP address. So it will be sharing the details to our application like this. So our application will be contacting this broker using this information.
