# How to Override Broker-Level Properties

This will be a very quick lecture. In the previous lecture, we were talking about some of the server properties which we will be using to set up the Kafka cluster, right? So you can provide those server properties as environment variables if you want. In that case, we have to follow some rules. So that is what this lecture is about. This will be particularly useful when you use Docker containers or Kubernetes to set up the Kafka cluster. The environment variable name should be starting with a Kafka underscore. Every single environment variable, this is how it has to start. Then you have to provide the property name in uppercase, replace all the dots with underscores. That's it. Three simple rules and you can provide the value as it is. For example, node ID is one server property, we know that, right? So if you are going to use environment variable, then this is how you have to provide Kafka underscore node underscore ID like this. So if you have listeners, then you have to provide like this Kafka underscore listeners. So if there is some property called a.b.c, just an example, you have to provide like this Kafka underscore a underscore b underscore c, okay? So this is how we will be providing environment variables to override some server properties.
