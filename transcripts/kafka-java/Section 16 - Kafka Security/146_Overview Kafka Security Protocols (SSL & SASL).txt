# Overview: Kafka Security Protocols (SSL & SASL)

In this section, let's talk about Kafka security. So far, our producer consumer applications were connecting to the Kafka broker using the plain text mechanism. That is, we did not enable authentication. Also, we did not encrypt the communication. In this section, our goal is to see how to enable authentication. So that these applications use some credentials to connect to the Kafka broker. Also encrypt the communication. Just a quick note that in the real life, all these things will be set up by your Kafka administrator. We as developers, we will be providing the login credentials in our application to connect to the Kafka broker. So I'm going to explain things in a high level. When we were talking about Kafka cluster, in that section, I had mentioned that Kafka has different security protocols. By default, this is what we have been using. No authentication, no encryption. So when we want authentication, we have two options. Sassel plain text, Sassel SSL. So when we use this option, we have authentication, but there is no encryption. When we use this, we have authentication, also encryption. So let's take a look at these two protocols one by one. What is this Sassel? Sassel stands for Simple Authentication and Security Layer. In simple terms, it is a standard way for a client and a server to do authentication. It's nothing new. It's been there for decades. Sassel separates the authentication method from the protocol. That means the application protocol does not need to know how authentication works internally. So the application protocol can support different authentication mechanisms like simple username, password, Kerberos, OAuth, etc. It's used by many applications. Messaging systems like Kafka, RabbitMQ use it. Email systems like SMTP and IMAP use it. And even some databases like Postgres use this. Now let's talk about JAZ. JAZ stands for Java Authentication and Authorization Service. In simple terms, using JAZ, Java applications provide login details. Kafka is written in Java, so it uses JAZ to get the credentials needed for Sassel authentication. Just to be very clear, Sassel is the authentication framework. Using JAZ, we supply the username, password, or other login details to Kafka. We can provide the JAZ configuration using a file or via application properties. We will see soon.
