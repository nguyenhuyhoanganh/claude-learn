# Netflux Overview: Building a Real-Time Recommendation Engine

In this section, we are going to build our final project, Netflix. This is something similar to the other project we built in my AWS course, if you had enrolled. The goal here is to build an application with multiple microservices, with event-driven communication, messaging with Kafka. It will have three services, a movie service that manages movie data, a customer service that manages customers and their preferences, particularly customers' favorite genre like action, adventure, etc. A recommendation service that will be responsible for generating movie recommendations and notifying the customers in real time. These services will communicate using events, not direct HTTP calls. Let's take movie service. It will have couple of REST APIs. It's mainly responsible for managing movie data. Whenever we add a new movie, it will be emitting movie added event like this with some basic information about the movie. Hey, this is the movie I just added. It notifies like this. Our recommendation service will be interested in consuming these events as it could recommend our customers that, hey, we have some newly added movies. Similarly, our customer service will have couple of REST APIs. It's mainly responsible for managing customer data. Whenever customers change their preferences like changing their genre, it will be emitting customer genre updated event. Our recommendation service will be interested in consuming these events as well as it could generate some personalized movie recommendations based on customers' preference. Our recommendation service will have one streaming endpoint to push the recommendations real-time. That is, whenever we add a new movie here, the movie service will be emitting this event. Recommendation service will be consuming that event and it will send a notification, real-time notification to the user that, hey, we have some newly added movie. Similarly, when customers change their preference, this will be generating one event. Recommendation service will be consuming that event and it will provide personalized recommendation via the streaming endpoint in real-time. So, this is very, very high level.
