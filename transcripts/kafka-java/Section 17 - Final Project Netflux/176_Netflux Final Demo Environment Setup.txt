# Netflux Final Demo: Environment Setup

At this point, we have completed our application and we have written the integration test to validate the APIs and the events etc. They all seem to be working fine. Now let's run our application so that we can test end to end. These are the ports we are going to use. The recommendation service will be listening on port 8080, customer service will use 6060 and the movie service will be listening on port 7070. I have already included one index.html as part of the recommendation service. So we will be accessing that to test our application. These are the steps. First, ensure that the Kafka broker is up and running. Then start customer service. Then start movie service. As part of the movie service, we have the movie data initializer. So for that, we have to ensure that this property is set to true. That movie data initializer will be sending one movie added event every three seconds. So let it run for some time. Then let's start the recommendation service. It will be consuming all the movie added events and it will be updating its movie table so that we will have something to see on the UI when you access our Netflix site using localhost colon 8080. That time, whenever it consumes the movie added event, we will be getting the real time streaming notification. Here you can also change your favorite genre. So when you change the genre on the UI, so that time we will be sending one HTTP request to customer service to update the genre. This will be triggering one event that will be consumed by the recommendation service and it will be providing the personalized recommendation immediately, which we can see on the UI. When you click on the movie images, that time it will be sending the movie details request to movie service. So we will be getting the detailed information about that movie, which we can also see that. Since we have multiple microservices and they all use different ports, I would have added this annotation on the rest of controllers. If you notice it's enabled for local testing because we have multiple microservices and they all use different ports. The browser will not work or we cannot send the request to multiple microservices by accessing the index HTML from the recommendation service. So this is why I am doing this way. Again, this is fine just for local testing.
