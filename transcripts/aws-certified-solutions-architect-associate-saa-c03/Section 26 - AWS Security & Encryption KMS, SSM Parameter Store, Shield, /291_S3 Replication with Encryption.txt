# S3 Replication with Encryption

So now let's talk about S3 Replication and its relation to encrypted objects. So if you enable S3 Replication from one bucket to another, then any unencrypted objects and objects encrypted with SSE-S3 will be replicated by default. Objects that are also encrypted with SSE-C, with the customer provided key, can also be replicated, and then we have objects encrypted with SSE-KMS. So by default, they're not replicated but we need to enable the option to actually replicate these objects. And so we specify with which KMS Key we want to encrypt the objects within the target buckets, and then we adapt this KMS Key Policy for the target key and we create an IAM Role that is allowing the S3 Replication service to first decrypt the data in the source bucket and then re-encrypt the data in the target bucket with the target KMS Key. And so when you do so, this enables replication because there is a lot of encryption and decryption happening. You may get KMS throttling errors, in which case you need to ask for a service quotas. So the question you may have is should I use multi-region key with S3 Replication? And the documentation says that you can use multi-region key for S3 Replication but currently they are treated as independent keys by the Amazon S3 service, and therefore the object is still going to be decrypted and then encrypted using the same key even though the key is a multi-region key, okay?
