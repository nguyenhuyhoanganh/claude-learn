# AWS App Runner

So now let's talk about the AWS App Runner service. It's a fully managed service that makes it easy to deploy web applications and APIs at scale and with this, anyone can deploy to AWS. You don't need to know anything about infrastructure, for containers or your source code or whatever you want. So the idea is that you start with your source code or a Docker container image and then you're going to configure the settings. So it shows how many VCPS you want, how much memory you want for your containers, if you want autoscaling and a few health checks. So some basic settings to define your web application or your API, and then automatically, that's it, it will start building and deploying the web app using the App Runner survey. So the container will be created and deployed, and then your API or your web app is deployed and you can access it right away using a URL and so, as you can see, there was no knowledge whatsoever of what goes on underneath. Underneath, of course, uses the AWS services, but as a user, you just don't know about it. So it's very quick. So with App Runner, you get a lot of goodness, you get automatic scaling, it's highly available, you get load balancing, you get encryption and your application, your container can also get access to your VPC. That means that you can connect to your databases, your cache, as well as your message queue services. So the use case for this is to very quickly deploy web apps, API, microservices, and do rapid production deployment with best practices, all using the AWS App Runner service. It's a very simple service. It does very simple things, but it's very powerful. Okay, that's it, I hope you liked it and I will see you in the next lecture.
