# VPC Flow Logs

So now, let's talk about VPC Flow Logs. VPP flow logs allow you to capture information from IP traffic going into your interfaces. So that could be at the VPC level, the subnet level, or the elastic network interface, ENI level. So we have three kinds of flow logs. They're very helpful to monitor and troubleshoot connectivity issues happening within your VPC. So these flow logs can be sent to Amazon S3, CloudWatch Logs, and Kinesis Data Firehose, and they will capture information for also AWS managed interfaces such as ELB, RDS, ElasticCache, Redshift, Workspaces, your net gateway, your transit gateway, and so on. So this is what a VPC flow log looks like. So there is a format associated with it, but there is a version, account ID, interface ID, source address, destination address, source port, destination port, protocol packets by start and action and log status. So this is metadata about the network packets going into your VPC. And you don't need to remember this at a high-level, right? But let's have a look at what information we can get out of these flow logs. So the source address and the destination address help identify problematic IP. This is where you see if an IP is repeatedly being denied, maybe there is something wrong with that IP, or maybe you're being attacked by a specific IP, source port and destination port helps you identify the probiotic ports. Action is going to be either accept or reject. And so it's going to say whether or not it's a success or failure at the SG or NACL level. And we'll see this in the very next slide. And so this can be used the VPC logs to do analytics on usage patterns or detect manage behavior, port scans, and so on. Now, to query these flow logs, you have two ways of doing it.
