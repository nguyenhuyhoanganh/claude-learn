# SSM Parameter Store Overview

So now let's talk about the SSM Parameter Store. It's a secure storage for your configuration and your secrets. And optionally, you can choose to encrypt these configurations and therefore make it secrets by using the KMS service. SSM Parameter Store is serverless, it's scalable, it's durable, and the SDK is very easy to use. On top of it, if you update your parameters then you have version tracking of them. The security is provided through IAM and you get notifications with Amazon EventBridge, in certain amount of cases. You have full integration with CloudFormation. That means that CloudFormation can leverage the parameters from parameter store as input parameters for your stacks. So let's take an example. We have an application and then we have the SSM parameter store. So we can store plain text configuration in this way. Then the IAM permissions of your applications are going to be checked. For example, your EC2 instance role or you can have encrypted configuration. In that case the SSM parameter store is going to encrypt it with KMS. And so the KMS service is going to be used for encryption and decryption. And of course, you need to make sure that your applications do have access to the underlying KMS key to perform that encryption and that decryption. So you can store parameters in the parameter store with a hierarchy. So for example, you can define my department as a path and underneath the My-app and then the Dev and then we have the Dev DB-URL and then the DB-password within that folder. So that means that your parameters goes all the way down into the hierarchy. We can go one level up and store a parameter for the Prod DB-URL, as well as a Prod DB password and go up to another app or another department. That really allows you to organize your parameters the way you want in a structured fashion. And then this is going to simplify your IAM policies to allow applications to have access to an entire department, or to an entire app, or just to an app department environment specific path. You also have the opportunity to access Secrets of Secrets Manager through the Parameter Store by using this reference right here. So it's a little trick that not many people know. And there are something called Public Parameters that are issued by AWS that you can use. For example, if you want to find the latest AMI for the Amazon Index 2 in your specific region. This is something that's available within the Parameter Store as an API call. So if you take an application, for example our Dev Lambda function is going to have an IAM roll that allows it to access the DB-URL and DB-password within the Dev of my app. And then if you have a Prod Lambda Function, thanks to again, a different IAM policy and maybe some environment variables, you can access the Prod DB-URL and Prod DB-password of another path.
