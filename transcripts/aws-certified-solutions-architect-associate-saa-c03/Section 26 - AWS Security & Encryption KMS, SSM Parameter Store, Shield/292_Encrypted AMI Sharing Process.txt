# Encrypted AMI Sharing Process

So here is a question that can come up in the exam. This is around the process to share an AMI with another account, and the AMI has been encrypted with a KMS key. So the AMI is in your source account, and it's encrypted with your KMS key. So how do you launch an EC2 instance in account B from the AMI in account A? So first you must modify the AMI property with a launch permission, and the list launch permission allows account B to launch this AMI. So effectively, this is how you share an AMI. You must modify the launch permissions and add the specified target to its account ID. Then you need to also share the KMS key for the accounts B to be able to use it. So this is done usually with a key policy. Then in account B, you would create an IAM role or IAM user with enough permissions to actually use both the KMS key and the AMI. So you must have it on the KMS side, access to the Describekey API call, the ReEncrypted API call, CreateGrant and Decrypt API calls. And then once all of this is done, you can simply launch an EC2 instance from that AMI and optionally the target accounts can choose to re-encrypt everything with the KMS key that it's owned, that it owns in its own accounts, to re-encrypt the volumes. But there you go, you can launch an EC2 instance. And so if you understand this process you're good to go to maybe answer one question at the exam.
