# AWS Secrets Manager - Overview

So now let's talk about a very simple service called the AWS Secrets Manager. So it's a newer service and it's meant for storing secrets and it's going to be different from the SSM parameter store, because on Secrets Manager, you have the capability to force the rotation of secrets every X number of days, and therefore you have a better secret management schedule. On top of it, from within Secrets Manager, you can force and automate the generation of secrets on the rotation. And for this, you have to define a Lambda function that will generate that new secrets. Moreover, Secrets Manager is really well integrated with different services on AWS. And I just showed you Amazon RDS for example, for MySQL, PostgreSQL, SQL or Aurora. But there are other services as well with AWS, other databases, that are integrated with Secrets Manager out of the box. That means that the username and the password to get into your database is stored directly in Secrets Manager and it can be rotated and so on. Now, Secrets can be encrypted using the KMS service. And so anytime in the exam you see Secrets, or integration for RDS, or for Aurora of Secrets, really think Secrets Manager. There's one more feature we need to know about which is the concept of multi-region Secrets. So the idea is that you can replicate your Secrets across multiple AWS regions and Secrets Manager Service will keep readers in sync with the primary Secrets. So here have two regions. We create a Secret in the primary one and it gets replicated as a same Secret into a secondary region. Now, why would we do this? Well, multiple things. Number one, in case there is a problem with US East 1, you're able, for example, to promote a replica Secret as a standalone secret. And then thanks to the fact that that Secret is replicated across regions, then you can build multiple regions apps. You can also have disaster recovery strategies, or if you have an RDS database that is also being replicated from one region to the next, then you can use the same Secret to access, the same, the RDS database, the corresponding one, in the corresponding region.
