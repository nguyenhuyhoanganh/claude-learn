# SQS - FIFO Queues

So now let's talk about Amazon's SQS FIFO queues. So FIFO means first in, first out, and it corresponds to the ordering of messages in the queue. That means that when you have a producer that send messages in a specific order, then one, two, three, four, then the consumer's polling messages from your SQS FIFO queue is going to read these messages in the same order. So this is a guarantee of the FIFO queue but this is not true for just a regular SQS queue. Messages can be received out of order. So because you have this ordering guarantee, you have limits on throughput, about 300 messages per second without batching, and 3,000 messages per second with batching. FIFO queue also have the ability to do something called exactly-once send. That means that any duplicates can be removed directly at the queue level. For this, you're just going to provide on every message you send a deduplication ID. And if this ID is seen twice in the time range of five minutes, then duplicates are going to be removed. So this is a very handy thing if you want to make sure you have no duplicates within five minutes. The messages are going to be processed in order by the consumer and the ordering guarantee is at the message group ID level. So you need to send a message group ID every time you send a message to Amazon SQS FIFO queue, and then all the messages are going to be in order for that specific group ID. So that's it.
