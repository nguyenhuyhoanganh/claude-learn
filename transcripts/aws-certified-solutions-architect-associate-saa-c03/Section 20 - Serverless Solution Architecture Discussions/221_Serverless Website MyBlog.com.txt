# Serverless Website: MyBlog.com

Okay, so let's talk about a serverless hosted website, maybe called MyBlog.com. So our website should scale globally, and we rarely write blogs, we often read blogs. So our blog is seen by hundreds of thousands of people online, and we really add blogs maybe one a day, one a week. But, most of the time, these blogs are being read. And so most of my websites is going to be purely static files, and maybe a little bit of my website is going to be a dynamic REST API. I want to implement caching, where possible, to really save cost and save latency, and have a great user experience. And any new user that subscribes to my website, to my blog, I really want them to receive a warm welcome email. And this should be serverless. And any photo uploaded to the blog, I also wanna have a thumbnail being generated, also serverless, because I really like serverless. So how do we implement all these requirements? Number one, we wanna serve content, it's static and it's global. So if you remember, we have our clients, and our static content maybe stored in Amazon S3. So how do we expose that bucket? Remember the Amazon S3 bucket is in specific region. How do we expose this globally? Well, we can use Amazon CloudFront. And Amazon CloudFront is a global distribution CDN, and so basically our client is going to interact with edge locations of Amazon CloudFront, and it's going to cache data coming straight from Amazon S3. Okay. Super easy. We've seen CloudFront, we've seen S3 is a classic architecture. Now how do we do this securely? Now, that's a very common question as well. So we have the clients, it's interacting with CloudFront and it's a global distribution still. But now we will add an Origin Access Control which will ensure that our Amazon S3 bucket can only be accessed by CloudFront. For this, we will add a bucket policy to only authorize the CloudFront distribution. Therefore, a user trying to access our Amazon S3 bucket directly will not be authorized, and we have secured our Amazon S3 buckets. Okay, so now we have this, and this is really good. How do we add a public serverless REST API? Well, for this, we'll have a REST HTTPS, talking to Amazon API Gateway, invoking a Lambda function, maybe querying and reading from DynamoDB. So we'll have a DynamoDB stream being created, and that DynamoDB stream will invoke a Lambda function. That Lambda function is going to be very special. It's going to have an IAM role, which allows us to use Amazon SES. So we haven't seen what Amazon SES is, but it's really simple. It's called Amazon Simple Email Service. So SES, and it basically allows us to send emails. So here our Amazon Lambda function can use the AWS SDK to send emails from Amazon SES. And here we go. We have a basically serverless User Welcome email flow, and really simple, no infrastructure to manage, it just works and scales really, really well. Okay, so now we said, okay, if users upload images, we want thumbnails to be created.
