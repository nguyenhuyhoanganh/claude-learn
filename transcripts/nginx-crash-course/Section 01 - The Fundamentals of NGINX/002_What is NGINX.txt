# What is NGINX?

What is nginx? Nginx is can be looked at as a web server. And when we say web server, it means it serves web content. And you can look at this content as a static content or even dynamic content, right, with CGI and CGI or content like that. But regardless, it it is listens on an Http endpoint and understand how to talk this Http protocol. And as a result it serves this web content for you. Right? And a lot of people use it just for that. But Nginx is also a reverse proxy. What does that mean? That means you can make it facing the internet and then in the back end it will do. It will take the request that it takes from the internet and then move them across your back end appropriately. For instance, you can use it for a load balancing. So everybody is hitting Nginx, everybody is talking to one thing, but Nginx cannot possibly answer all these requests, so it will load balance the requests into multiple backends. So that's one use case for a reverse proxy. Another use case for reverse proxy is back end routing. I want to route if you're going to slash app one, go to these sets of server.
