As of now you can see this is a empty() method. On top of this method we need to mention an annotation which is @EventSourcingHandler because we need to tell to the framework whenever these customerCreatedEvent is published from the Aggregate class. We want this method to catch that event and store that data into the write database. Once this writing of the data into the right database is completed, the framework is going to take care of publishing this event further into the event bus. Inside this method, the logic that we need to write is going to be very simple. We just have to write this code and behind the scenes what framework is going to do is it is going to look for the storage database and inside the storage database all these event details will be stored in an historical or in an sequential style. Long back we discussed on how the event sourcing storage is going to happen. At the very bottom, you're going to have the very first event. So whenever the customer is being created similarly, at the very bottom, the very first event is going to be created. Similarly, when the updates are happening to this customer, all the subsequent events that are going to be right on top of the existing events. At any point of time if you look into the storage database, we are going to have all the sequence of events that are happened or on top of a given customer. That's the advantage of Event sourcing pattern. But here you may have a question which is, as of now, we have not configured any write database for our Axon framework to store the customer data.
