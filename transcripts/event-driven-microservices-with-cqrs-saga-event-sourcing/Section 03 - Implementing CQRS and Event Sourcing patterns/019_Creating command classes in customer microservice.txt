Whenever someone want to invoke a command operation, they are going to invoke the Rest API that we are going to expose as part of the command component. From end user perspective, they are always going to send the input in a JSON format. Once we receive the input data, we can perform the validations and once we are happy with the input validations, we need to initiate a command and this command will be processed by the axon server. During the process of this command, the axon server, it is going to store the data received inside the right database. Once the right database updation is completed, the axon server it is going to trigger the event and this event will be consumed by the query side. And on the query side, this event is going to be processed and the data is also going to be updated on the read database as well. To make this entire flow work without any issues, we need to let Axon server and Axon Framework know on what commands needs to be trigger, or on what events needs to be trigger when a specific API is invoked inside our microservice.
