# Performing validations using MessageDispatchInterceptor implementation - Part 2

We have our Interceptor class ready. But if you try to test this application, the interceptor logic is not going to be invoked. To make this work, we need to make one more small configuration. Let me go to the Spring Boot main class which is CustomersApplication. Inside this class I'm going to create a method which is public void method name I'm going to mention as registerCustomerCommandInterceptor(). And to this method I'm going to pass two input parameters. The very first one is, ApplicationContext and the second one is the CommandGateway. So as of now we are dispatching all the commands from the Controller class with the help of commandGateway bean only. That's why we need to pass CommandGateway as an input to this method. On top of this method I'm going to mention @Autowired annotation. Now using this CommandGateway object or bean, I need to invoke a method which is registeredDispatchInterceptor() . To this method, I need to pass the bean of my Interceptor class. So how to get the same using the Context.getBean(). And to this get bean method we can pass the class name which is CustomerCommandInterceptor.class. Once you added these annotation we should be good. Our interceptor is going to be registered with the CommandGateway. Now anytime if you try to dispatch a command with the help of CommandGateway, this interceptor logic is going to be executed. Let me save the changes do the build. Once the build is completed, I'll try to restart my customer microservice so that all our changes will be reflected properly. Once my customer microservice is started successfully, we can go to the CustomerCommandInterceptor and we can try to keep a breakpoint here inside the lambda expression. Now from the postman, I'll try to invoke some operations first what I'll try to do is, I'll try to create a customer by using the same mobileNumber.
