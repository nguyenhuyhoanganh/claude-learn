# Axon Framework IntelliJ plugin

There is a IntelliJ plugin related to the Axon framework, which you can install inside your IntelliJ, and using this plugin, you can improve the productivity while you are trying to build microservices with the help of Axon Framework. Let me show you how to install this plugin and how to use it inside your IntelliJ. First, you need to go to the plugins inside your IntelliJ. Under the plugins, you can go to the marketplace and simply search for Axon. So there is a plugin with the name Axon Framework. Since I already installed it is showing me disable option here. But for you, since this plugin is not installed, there should be an install button. Please go ahead and click on the same. I know you might be using lot many other plugins and your IntelliJ IDE also might be crying. How many more plugins I need to accept? Sometimes we can't avoid plugins because they make our life so easy. I hope your installation is successful. Once the installation is completed, you need to restart your IDE once the plugin is installed successfully you can go to the classes where we have implemented Axon framework related components like command, handlers, event handlers for example inside Customer microservice. Inside the commandController package, there is an class with the name CustomerCommandController. If you can open this inside all these APIs, we are trying to create a command. For example, here I'm trying to create the object of createCustomerCommand. The same applies for updateCustomerCommand and deleteCustomerCommand. If you closely observe in the line where I'm trying to build the command object, there is a left side arrow with the orange color. So just after the line 31 you will be able to see there is an arrow. So if you click on this arrow, it is going to take you to the method where a given particular command is being handler. Since the createCustomer command is being handled inside the CustomerAggregate, we are landed onto this constructor. Similarly, in case if you want to know from where this command is being triggered, you can click on this arrow which will take you to the actual Controller class. Let me click on this arrow one more time. And now we are inside this commandHandler() method. From the commandHandler() method what we are going to do we are going to execute some business logic followed by we are going to trigger the event here. Inside this method I'm trying to create an object of CustomerCreatedEvent. The same we are going to publish to the event bus. If I try to click on this arrow, you will be able to see this time I have two options. The plugin is showing that this event is being handled at two different places. The very first one is inside the CustomerProjection class, followed by the second one is inside the CustomerAggregate class with an annotation @EventSourcingHandler. Let me select this event sourcing handler option, and it is going to take me to the method where we are handling these events. With the help of @EventSourcingHandler annotation. We can also click on this arrow which will take us to the constructor where we are trying to create this event. Let me try to click on this arrow one more time. This time I'm going to select the CustomerProjection option. As you can see, we are landed onto the method where we are trying to handle these customerCreatedEvent. This way, you can always use this plugin to navigate from commands to events, events to command. Usually inside real applications there will be too many commands and events. Instead of opening all these methods classes manually, we can leverage these arrows which are going to improve our productivity. I hope you are happy with this plugin. Unfortunately, this plugin as of now it is available only inside IntelliJ marketplace but not in other IDE marketplace like eclipse Visual Studio Code. You can't find this plugin. I'm not sure if the Axon Framework team is going to publish those plugins in future, but for now, if you are using IntelliJ for your day to day development, you can happily use this plugin. Thank you and I'll catch you in the next lecture, bye.
