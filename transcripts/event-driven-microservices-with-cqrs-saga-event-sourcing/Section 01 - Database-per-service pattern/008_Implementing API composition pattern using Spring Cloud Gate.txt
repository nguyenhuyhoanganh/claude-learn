Here we are trying to use the mono as a return type. If you're not clear about the reactive  programming model, let me spend some time explaining about the purpose of mono. As you know, Spring Cloud Gateway is built based upon the reactive programming model, just to make sure it handles a large number of requests with less number of threads and memory. Inside react two programming model we have two different concepts one is mono and the other one is flux. Whenever you are expecting a single object as a response, then you can use the mono, whereas if you are expecting multiple objects or collection of objects, you can use the flux. So just like how we have mono, we also have flux. But since we are expecting only a single customer DTO object, I'm trying to use mono. With this mono what I'm trying to tell to the Spring Reactive  programming model is, I want to invoke this API asynchronously in a non-blocking style. As soon as this API is invoked behind the scenes, the thread will not wait for the response. The thread will move on to the next piece of code. This way we can parallelly try to invoke all the microservices involved and towards the end, we can try to wait for the response from all the microservices and compose the response.
