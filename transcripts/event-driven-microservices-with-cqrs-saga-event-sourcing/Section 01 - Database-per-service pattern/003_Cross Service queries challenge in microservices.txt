# Cross Service queries challenge in microservices

The very first challenge that organizations are going to face when they adopt database per service pattern is Cross-service Queries. Let's try to understand in detail about this challenge. Let's imagine a bank application. Inside the bank there can be multiple microservices. As you can see on the screen we have customer microservice, accounts microservice cards and loans microservice. In total, we have four different microservices having their own database. In this kind of scenario, let's imagine this bank has an UI application. Inside this UI application, there can be multiple pages where the data from the multiple services needs to be displayed. Let's consider there is a page with the name profile, under this profile the bank UI application it want to display the data from the customer, accounts, loans and cards. Basically, this profile web page will have the summary of the data belongs to an end user. In this kind of scenario, if your bank application is a monolithic application, it's very easy and straightforward to display this summary of data. Because inside the monolithic applications we're going to have a single database. Though this single database can have multiple tables storing all this information. What developers they will do. They're going to join all these tables data by using inner joins outer joins. And the result they can send to the UI application. But with microservices using individual databases it is not going to be easy. We can't perform the join operations because all these are individual databases which are deployed at multiple locations. I hope you are getting the challenge here. As you can see, the challenge name is Cross-Service Queries. Whenever you have a scenario where you want to fetch the data from multiple microservices, then you will end up querying multiple databases which are owned by multiple microservices. Let's try to understand how to overcome this challenge or how to handle this scenario inside any enterprise application. Cross-Service Queries challenge can be addressed by using two different patterns. The very first one is API composition pattern and the second one is CQRS pattern. For now, let's not worry about the CQRS pattern because it's a complex pattern and it requires a dedicated section. We are going to discuss about this in the coming sections. For the next few lectures let's try to focus on these API composition pattern. By implementing this pattern inside our microservices, we are going to fetch the data from the multiple microservices. As the name itself is self-explanatory. What is this pattern is going to do? There will be a component which is responsible to invoke all the involved microservices to fetch the data. Once the data is individually fetched from the microservices inside this component, we are going to aggregate the response or compose the response as a single response, and the same will be sent to the client application.
