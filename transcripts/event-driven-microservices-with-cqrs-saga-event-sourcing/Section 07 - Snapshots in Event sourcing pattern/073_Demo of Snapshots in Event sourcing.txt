# Demo of Snapshots in Event sourcing

Every product or framework that supports event sourcing pattern it should also support the Snapchat functionality to overcome the challenge of performance in the scenarios of high volume of events. As of now, inside our microservices, we are leveraging the Axon framework to implement the event sourcing pattern. This Axon framework also supports the Snapchat functionality. If you try to look for the documentation of Axon Framework, you will be able to see a reference around the snap charts. If you are interested you can read this entire page. I'm also going to show you a small demo on how to create these snapshot and leverage inside our microservices. To create a Snapchat, first we need to create a bean inside our microservices. These beans should be of type Snapchat trigger definition. This is an interface inside Axon Framework, which is going to let you define on how the Snapchat needs to be triggered. If you look for the implementation of this interface, there are three implementation classes as of now, there is a class with the name no Snapchat trigger definition. If you are looking for an option where at any case, you don't want to create the Snapchat even in the future. In such scenarios, you can leverage this no Snapchat trigger definition, but if you are looking for an option to configure Snapchat, you have two different options. The first option is based upon the event account or based upon the number of events you can create a Snapchat. For example you can configure a bean with the help of this class by conveying for every 100 events I want, a Snapchat needs to be triggered or for every ten events I want, a Snapchat needs to be triggered. So such configurations we can make with the help of event count to Snapchat Trigger definition. The other option is based upon the Aggregate load time you can define a definition. We know the aggregator it is going to load all the previous events. It is going to replay all the previous events. If this entire replay operation, if it is taking more than certain amount of time, then you can consider that as a triggering point to create your Snapchat. If you open this class, you will be able to see the documentation. A Snapchat trigger mechanism which, based on the loading time of an aggregate, decides when to trigger the creation of a Snapchat.
