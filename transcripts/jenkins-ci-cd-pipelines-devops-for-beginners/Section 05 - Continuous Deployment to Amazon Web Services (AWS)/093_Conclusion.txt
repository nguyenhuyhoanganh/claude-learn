# Conclusion

In this section, we have learned some fundamental AWS concepts and created a fully automated pipeline that deploys our website to the AWS cloud. How cool is that? During this process, we have covered two important AWS services. S3, a file storage service, and IAM, a service used to control access to AWS resources. In the process. We have configured S3 to function as a web service displaying our static website. For simple use cases, this setup is quite effective. If you want to use this for your own projects, you just need a domain name and you can link that with your S3 bucket. This way you avoid using the long and complex URL provided by AWS. In this process, we have also learned how to use the AWS cli to interact with different AWS services. And at the same time, hopefully you have understood how important it is to spend that time and to go over the command reference over the documentation to understand how to use things.
