# Introduction to cloud computing

In this section, we will learn about deployments and will take our website project and deploy it to the Amazon Web Services Cloud or simply AWS. Along the way, we will learn how to manage cloud services using a CLI tool and in the process, discover some other useful Jenkins features. But why use the cloud in the first place? Just because it is what everyone is using? Cloud computing is popular for several reasons. First, it provides scalability. This means that you can easily increase or decrease your resources based on your needs without having to invest in physical hardware. For example, if your website suddenly gets a lot of traffic, the cloud can automatically provide more resources to handle the load, ensuring your website remains fast and responsive. Second, cloud computing is cost effective. You only pay for what you use, which can be much cheaper than maintaining your own servers. For instance, instead of buying expensive servers and paying for their maintenance, you can use cloud services and pay only for the resources you consume. What good does it do if you have a powerful server, which can handle the traffic that you're getting when you're running the Black Friday sale, if for the rest of the year it is only used at 20% of capacity. Finally, the cloud offers reliability and security. Cloud providers like AWS, Amazon Azure, and Google Cloud have data centers around the world with robust security measures. This means that your data is safe and your services are available 24 over seven. Even if there is a hardware failure or other issues in one location. While there are certainly many situations and reasons when using the cloud does not make sense, as a DevOps professional, you need to know how to use at least one cloud provider. By the end of this section, we will have a complete CI CD pipeline that will take our website, build it, test it, and let it flow in the AWS cloud. While this section shows AWS as an example, again our focus is on understanding the principles. You can take what you have learned here and easily apply it to Microsoft Azure, Google Cloud Platform or other providers.
