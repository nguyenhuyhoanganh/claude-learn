# Section Overview

As we have constructed this pipeline over the past sections, we had a constant need for installing additional tools. For example, at one point we needed the JC library to be able to parse a JSON file. For that, we found an npm package called node JC, which was convenient since we were using a Docker image that already had Node.js installed. As all project was using it. But what if we work on a Python application? Does it mean that we need to install node js as well? The answer is no. Definitely not. This is not how we want to approach things. So in this section we will explore approaches for finding appropriate Docker images for our projects. Understand how to install additional tools, and even learn how to build a custom Docker image and then use it in the pipeline. But first, let me give you a bit of a historical context. Maybe 15 years ago or so, most software applications were big monoliths characterized by long release cycles and very risky deployments. Most of the time after the development team finished their work, they would hand a software package over to the operations team, which would handle the deployment and monitoring, usually after an okay from the quality assurance team. Even for a small change in one of the components of the monolith, the entire system needs to be deployed as a whole. Having a release only once per month was not unusual. Nowadays, new software applications are broken down into smaller components and are generically referred to as microservices. Smaller software components are most of the time decoupled from other components and can be developed, tested, and deployed independently of others. This shift has not only changed how software is built, deployed and maintained, but also has allowed organizations all around the world to bring features to market with a speed never seen before. This is why nowadays, the DevOps practices discussed in the course enable us to deploy IT confidence even multiple times per day. One of the underlying technologies that have dramatically contributed to the shift has been Docker. Docker has allowed us to package these smaller applications into containers and deploy them with ease. But there are also many benefits to using Docker in Jenkins. So in the beginning of this section, we will explore a few Docker fundamentals to ensure we are on the same page. And after that, we'll understand how we can leverage Docker while using Jenkins. Let's again.
