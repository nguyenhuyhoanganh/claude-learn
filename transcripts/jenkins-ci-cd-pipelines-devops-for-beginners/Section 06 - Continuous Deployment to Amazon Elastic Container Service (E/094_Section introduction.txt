# Section introduction

Welcome to this new section of the course where we will learn some new AWS services. Up to now, we have successfully deployed a static website to AWS using the simple storage service known as S3. This method is one of the most straightforward ways to use Jenkins along with AWS. However, today's applications are generally more complex and the reality is that many of them use Docker. So in a nutshell, we will dockerize our application and deploy it to AWS. But what does it mean to dockerize an application in the first place? Dockerize means to package an application along with all its dependencies into a container. A container is like a small package that contains everything an application needs to run. This makes it super easy to move the application between different environments without any issues. So in this section we will dockerize our website which involves building a Docker image. This is like creating a blueprint of our container which to be functional, needs a web server and our website files. Then we will store our Docker image in a Docker registry on AWS using a service called Amazon Container Registry. Finally, we're going to take that image from the Amazon Container Registry and deploy our Docker container using Amazon Elastic Container Service, a service on AWS that allows us to run and manage Docker containers. And now it sounds a bit complicated, and the reality is that this section will increase the difficulty level quite a bit. There will be a lot of new concepts and moving parts, so it is normal to feel a bit overwhelmed at the start. That's just a sign you are learning significant new skills. We'll still tackle everything step by step to ensure that you can follow along comfortably. Apart from all this AWS stuff, to make our Jenkins pipeline more efficient, we will also learn some new Linux commands. So if you're excited to follow along and learn more, let's jump right into it.
