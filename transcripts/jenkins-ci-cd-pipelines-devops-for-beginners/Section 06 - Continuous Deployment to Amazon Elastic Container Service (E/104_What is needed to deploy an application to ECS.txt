# What is needed to deploy an application to ECS

Up to now, we have focused on understanding the basic steps needed to deploy an application to X, and for that, we have used a public Docker image containing the popular web server engine X as an example. But our goal is to deploy our own application, which we are building in the Jenkins pipeline. So to deploy our own application to X. There are still a few steps we need to take. First of all, X works only with Docker images. so we cannot simply upload some files and deploy them. What we need to do is package our website in a Docker image. We will do this by creating a docker file, which defines the steps needed to build the image that we need. Next we're going to use the command docker build, which will build a Docker image based on the docker file provided. But we are not done yet. The docker image that we have created is not a regular file or an archive that we can just declare as an artifact, or upload to S3 or somewhere else. Also, we cannot upload this image to ECS directly. Instead, we need to publish it to a container registry. Docker hub is a good example for a container registry, and while it also supports private images, it's typically used for public images. All the images that we have searched on Docker Hub so far, they are all public images. Nginx, which we have deployed, is a public image. For the sake of argument, we're just going to say that this is a private application, and we definitely do not want to publish it somewhere where anyone else could download it. And luckily, AWS has a service which is called Amazon ECR, which stands for Elastic Container Registry, which can be used as a private registry. And because it is an AWS service, the integration to ECS is much, much easier than using Docker Hub. So in a nutshell, we're going to create a Docker file. We're going to build an image based on the Docker file. And once we have that image we're going to push it to Amazon ECR our Docker container registry. And from there we can deploy it to ECS.
