# What is a shell?

Throughout the course we will be using Linux commands. We have already learned about a few commands like echo, create a directory, cat or touch. And we have put them in this ACH, which stands for shell. We typically type this commands through a command line interface or CLI. Sometimes we call it a console or a terminal or a shell. While technically speaking they are not the same, you may notice me and others use them interchangeably. A command line interface is the opposite of a graphical interface. If you're trying to create a new folder on your computer, you'll probably right click and choose create a Folder. And then you would open a text editor and put a file inside there, or something like that. Now this is using the graphical interface of the operating system. And the commands are the command line interface. So we work with a CLI to write commands. The computers we interact with have no user interface that we can use. So it's not about somehow getting Jenkins to right click on something and then create a directory. So we are using these commands to replace any need for a graphical user interaction. And anyway, automating graphical interfaces is not as easy or as reliable as using commands. This CLI interface that computers have is called the shell. So the shell is simply the outer layer of the system, and it is the only thing that we see from the outside. When we send these commands to the system, the system will take them and will run them. This is how we can interact with the system. With Jenkins, we essentially automate a set of commands which we execute in a particular order. For example, here, the order how we're running these commands is super important, where first creating a directory and then we're putting a file in that directory. If we are trying to do something else before that, these commands may fail. Through the course, I'm going to introduce you to more and more of these Linux commands, and I'm going to try to explain them to my best. But there's no replacement for trying a few things on your own. Please see the resources folder for ideas on how you can set up your own Linux environment on your computer, and use some of the Linux commands on your own so that you can gain more confidence. Anyway, let's go back to the course.
